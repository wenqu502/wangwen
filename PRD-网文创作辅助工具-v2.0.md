# 网文创作辅助工具 — 产品需求文档 (PRD) v2.0

> **版本**: v2.0  
> **日期**: 2026-05-03  
> **定位**: 面向网文作者的一站式创作辅助 SaaS 工具  
> **形态**: Web 应用（桌面端为主，支持大屏创作场景）

---

## 一、产品概述

一个面向网文作者的一站式创作辅助工具，以「角色 - 剧情 - 设定」三大支柱为核心，辅以 AI 智能校验与灵感管理，系统性地解决网文创作中「角色崩坏、剧情混乱、设定失控、卡文断更」四大顽疾。

**核心价值主张**：不让作者因为「忘记了自己写过什么」而写崩。

---

## 二、目标用户

| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| 职业/半职业网文作者 | 日更 4000~10000 字，有签约压力 | 防止写崩、提升日更效率、管理长篇设定 |
| 新人写手 | 刚开始创作，缺乏大纲能力 | 可视化引导、降低卡文概率、快速上手 |
| 同人创作者 | 基于原作进行二次创作 | 角色档案管理、剧情线梳理、人设不 OOC |
| 剧本杀/OC 创作者 | 注重人物关系与世界观 | 关系图可视化、设定库管理 |

**使用场景**：开书前的大纲规划期、连载中的设定查阅与维护、卡文时的灵感激发与续写辅助、完本后的资料整理。

## 四、核心功能模块

### 4.1 角色小像（Character Card）

为每个角色建立结构化档案，并生成视觉化形象。

#### 4.1.1 角色档案字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | 文本 | 本名、别名、绰号 |
| 阵营/势力 | 单选/多选 | 所属组织、门派、国家等，关联世界观设定库 |
| 身份标签 | 标签 | 如「男主」「反派」「工具人」「CP」 |
| 外貌特征 | 富文本 | 发色、瞳色、身高、特殊标记、常穿服饰 |
| 性格关键词 | 标签 | 如「傲娇」「腹黑」「温柔」「疯批」 |
| 性格详细描述 | 富文本 | 深层性格、行为模式、应激反应 |
| 能力/技能 | 富文本 | 武力值、特殊能力、职业技能 |
| 背景故事 | 富文本 | 出身、成长经历、核心创伤/动机 |
| 目标与弧光 | 富文本 | 角色在故事中的目标、成长变化轨迹 |
| 经典台词 | 文本列表 | 口头禅、名场面台词 |
| 关联角色 | 关系引用 | 链接到其他角色及关系类型 |
| 首次登场 | 章节引用 | 关联到剧情树中的具体节点 |
| 状态 | 单选 | 存活 / 死亡 / 失踪 / 封印 |
| 形象图 | 图片 | AI 生成或手动上传的角色立绘 |

#### 4.1.2 AI 形象生成

- 基于角色外貌描述 + 风格选择（古风/现代/西幻/赛博朋克等），调用文生图模型生成角色立绘
- 支持同一角色多服装/多时期的形象版本（如「前期平民」「后期黑化」）
- 生成结果保存到角色卡，作为写作时的视觉参考

#### 4.1.3 人设一致性校验（联动 AI 功能 6.1）

- 在作者撰写章节正文时，实时检测角色言行是否与档案中的「性格关键词」「性格详细描述」一致
- 检测到偏差时，侧边栏弹出提醒：「此处角色表现与档案设定不符，建议复核」
- 支持一键对比：弹出角色档案与当前章节的并排视图

#### 4.1.4 页面布局

```
┌─────────────────────────────────────────────────────┐
│  顶部：作品选择 | 保存状态 | 全局搜索框                │
├──────────┬──────────────────────────────────────────┤
│          │  Tab 切换:                               │
│  左侧     │  ┌────────┬────────┬────────┬────────┐  │
│  角色     │  │ 角色小像 │ 剧情分支树 │ 人物关系图 │ ... │  │
│  列表     │  └────────┴────────┴────────┴────────┘  │
│  (可搜索  │                                          │
│   /筛选)  │  [角色详情区域]                          │
│           │  ┌──────────────────────────────────┐   │
│  [+新建]  │  │ [头像/形象图] 姓名 + 标签           │   │
│           │  │ ┌──────┐                          │   │
│           │  │ │ 形象  │  基础信息 / 能力 / 关系   │   │
```

### 4.2 剧情分支树（Plot Branch Tree）

以树形/有向图的方式管理故事脉络，支持多主线、多分支、循环/回收结构。

#### 4.2.1 核心概念

| 概念 | 说明 |
|------|------|
| 节点 (Node) | 代表一个剧情事件/章节/情节点，包含标题、摘要、内容 |
| 主线 (Trunk) | 故事的骨干脉络，如「主线剧情」「支线A」「IF线」 |
| 分支 (Branch) | 从某一节点分出的平行剧情 |
| 连接 (Edge) | 节点间的因果关系或顺序关系，可标注条件 |
| 回收 (Merge) | 多条分支最终汇合到同一节点的结构 |
| 伏笔 (Foreshadowing) | 节点中埋下的悬念或线索 |
| 回收 (Payoff) | 后续节点中对伏笔的呼应与解答 |

#### 4.2.2 节点数据结构

```json
{
  "id": "node_001",
  "title": "第三章：宗门大比",
  "summary": "主角参加宗门大比，击败内门弟子，引起长老关注",
  "content": "完整章节大纲或正文...",
  "type": "主线|支线|IF线|伏笔",
  "status": "已写|进行中|待写|废弃",
  "characters": ["角色A", "角色B"],
  "location": "青云宗演武场",
  "tags": ["高潮", "打脸"],
  "parentIds": ["node_000"],
  "childIds": ["node_002", "node_003"],
  "condition": "角色A修为≥筑基期",
  "foreshadowing": [
    { "id": "fs_1", "desc": "神秘老者出现", "status": "未回收" }
  ],
  "payoff": [
    { "id": "po_1", "desc": "老者身份揭露", "linkedForeshadowing": "fs_1" }
  ],
  "wordCountTarget": 4000,
  "wordCountActual": 4200,
  "createdAt": "2026-03-01",
  "updatedAt": "2026-03-05"
}
```

#### 4.2.3 可视化交互

- **画布操作**：拖拽画布、缩放（Ctrl+滚轮）、框选多节点
- **节点操作**：单击查看详情面板，双击进入编辑，右键菜单（添加子节点/添加兄弟节点/删除/标记完成）
- **连线模式**：拖拽创建节点间关系，自动吸附对齐，连线可标注条件文字
- **筛选视图**：按主线/角色/时间线/状态过滤显示
- **时间轴视图**：线性阅读模式，适合回读剧情顺序
- **细纲模式**：每个节点可展开为卡片，显示细纲和正文片段

#### 4.2.4 伏笔追踪（联动 AI 功能 6.2）

- 节点可标记「埋下伏笔」和「回收伏笔」
- 伏笔-回收之间以**虚线**连接，点击可跳转关联节点
- **未回收伏笔高亮提醒**：超过 N 章未回收的伏笔，在画布上以红色闪烁标记
- 支持「伏笔回收率」统计面板：已埋 / 已回收 / 待回收

#### 4.2.5 卡文辅助（联动 AI 功能 6.5）

- 在节点编辑面板中，点击「AI 续写」按钮
- 系统根据当前节点内容 + 子节点目标 + 角色档案，生成续写建议/草稿
- 支持「生成三种走向」供作者选择

### 4.3 人物关系图（Relationship Graph）

以力导向图的形式展示角色间的关系网络。

#### 4.3.1 关系类型预设

| 关系大类 | 具体关系 | 连线样式 |
|----------|----------|----------|
| 血缘 | 父子、兄妹、师徒 | 实线 + 暖色（橙/红） |
| 情感 | 爱慕、情敌、挚友、宿敌 | 实线 + 图标（心形/闪电） |
| 利益 | 同盟、上下级、交易 | 虚线 + 冷色（蓝/灰） |
| 冲突 | 仇杀、竞争、背叛 | 粗线 + 红色 |
| 隐藏 | 真实身份、暗中操控 | 点线 + 灰色（可切换显隐） |

#### 4.3.2 交互功能

- **点击角色节点**：展开角色小像侧边栏，快速查看档案
- **点击关系线**：显示关系详情（建立时间、关键事件、当前状态）
- **圈层布局**：以核心角色为中心，按亲疏远近自动分层
- **势力着色**：同一阵营的角色节点颜色统一
- **动态演变**：按「卷/章节」滑动时间轴，观看关系网络随剧情演变（角色死亡后节点变灰，关系线变虚）
- **新增关系**：拖拽一个角色到另一个角色上，弹出关系创建弹窗

### 4.4 世界观设定库（Worldbuilding Library）【新增】

系统化管理小说的世界观设定，防止「设定爆炸」和「设定遗忘」。

#### 4.4.1 设定分类

| 分类 | 说明 | 示例 |
|------|------|------|
| 地理/地图 | 大陆、国家、城市、地形 | 「九州大陆」「青云山脉」 |
| 势力/组织 | 宗门、国家、家族、商会 | 「青云宗」「魔道联盟」 |
| 修炼体系 | 等级、境界、功法体系 | 「炼气→筑基→金丹」 |
| 种族/物种 | 人族、妖族、魔族等 | 「龙族：寿命万年，天生御火」 |
| 历史/年表 | 重大事件时间线 | 「天元历 1024 年：仙魔大战」 |
| 规则/法则 | 世界运行的底层规则 | 「筑基期不可御剑飞行」 |
| 道具/物品 | 法器、丹药、灵石等 | 「玄天剑：上古神器」 |
| 术法/技能 | 招式、神通、阵法 | 「九转金身诀」 |

#### 4.4.2 地图可视化编辑器

- 支持上传底图或空白画布
- 可在地图上标记地点（城市、宗门、秘境等），标记关联到设定条目
- 支持势力范围着色（如魔道势力范围涂红，正道涂蓝）
- 点击标记弹出设定详情卡片
- 支持多图层：地形层、势力层、事件层

#### 4.4.3 修炼体系编辑器

- 可视化编辑境界等级树
- 每个境界可配置：名称、特征描述、突破条件、能力变化
- 自动渲染为阶梯图/金字塔图
- 支持「分支体系」（如炼体流 vs 法修流的平行体系）

#### 4.4.4 年表/时间线

- 按时间轴展示世界重大事件
- 事件可关联到剧情分支树的节点（如「天元历1024年仙魔大战」对应小说第50章）
- 支持多时间尺度切换：纪元 → 年 → 月 → 日

#### 4.4.5 设定规则冲突检测（联动 AI 功能 6.3）

- 作者在设定库中录入「硬性规则」
- 在撰写章节或编辑剧情节点时，若内容违反已录入的规则，实时弹出冲突提醒
- 示例：规则库录入「筑基期不可御剑飞行」，作者在章节中写道「筑基期的主角御剑而去」，触发冲突警告

### 4.5 灵感便签（Idea Sticky Notes）【新增】

解决「灵感碎片化、容易遗忘」的痛点。

#### 4.5.1 功能设计

- **快速记录**：全局快捷键（如 Ctrl+Shift+I）唤出快速记录浮窗，输入后自动保存
- **便签列表**：按时间倒序展示所有灵感便签，支持标签分类（如「剧情」「角色」「台词」「设定」）
- **拖拽关联**：便签可拖拽到剧情分支树的节点上、角色卡片上、或设定库条目上，建立关联
- **待处理/已归档**：便签默认「待处理」，关联或消化后可标记为「已归档」
- **搜索**：全文搜索便签内容

#### 4.5.2 数据结构

```json
{

  "tags": ["剧情", "金手指"],
  "status": "待处理|已归档",
  "linkedEntity": {
    "type": "node|character|setting",
    "id": "node_003"
  },
  "createdAt": "2026-03-05T14:30:00"
}
```

## 五、AI 智能辅助功能

### 5.1 人设一致性校验

**触发时机**：作者在章节编辑器中输入正文时（实时），或提交章节时（批量）。

**校验逻辑**：
1. 识别当前段落中出现的角色
2. 从角色档案中提取「性格关键词」「性格详细描述」「经典台词风格」
3. 用 NLP/LLM 分析当前段落中该角色的言行、心理描写、对话语气
4. 比对档案设定，计算一致性分数
5. 低于阈值时，在侧边栏或段落旁标记提醒

**输出示例**：
> ⚠️ 角色「林云」一致性提醒  
> 档案设定：「性格沉稳、寡言少语、遇事冷静」  
> 当前段落：「林云激动地大喊：『不可能！这绝对不可能！』」  
> 建议：若为极端情境下的失态，可补充心理描写说明；否则建议调整台词风格。

### 5.2 伏笔追踪与回收提醒

**功能流程**：
1. 作者在剧情节点中标记「埋下伏笔」（文本高亮 + 添加描述）
2. 系统自动记录伏笔状态为「未回收」
3. 作者在其他节点中标记「回收伏笔」时，可关联到已埋下的伏笔
4. 系统维护伏笔列表，展示：伏笔描述、埋下节点、回收节点、状态
5. **提醒规则**：
   - 超过 10 章未回收的伏笔 → 画布高亮 + 列表置顶
   - 作品完结时仍有未回收伏笔 → 生成「烂尾风险报告」

### 5.3 设定规则冲突检测

**功能流程**：
1. 作者在世界观设定库 → 规则分类中录入硬性规则（支持自然语言描述）
2. 规则自动解析为结构化条件
3. 在剧情节点正文、角色能力描述、章节编辑器中，实时/批量扫描文本
4. 检测到疑似违反规则的内容时，弹出冲突提醒
5. 支持「例外声明」：作者可标记某处为「特殊情况，规则不适用」，避免误报

**示例**：
> ❌ 规则冲突 detected  
> 规则：「金丹期以下修士无法辟谷」  
> 冲突文本：「筑基期的李华闭关三月，不吃不喝」  
> 位置：第47章 第3段  
> [忽略此条] [修改文本] [添加例外说明]

### 5.4 智能起名助手

**支持起名类型**：
- 角色名（中文/西幻/日式等风格）
- 招式/技能名
- 地名（国家、城市、山脉、河流）
- 道具/法器名
- 组织/宗门名

**交互方式**：
1. 作者选择起名类型 + 风格标签（如「古风」「霸气」「带火属性」）
2. AI 生成 10 个候选名称
3. 点击「喜欢」可保存到「候选池」，点击「重生成」可换一批
4. 支持「以已有名称为参考，生成同风格系列名」（如生成青云宗七峰峰主的名字）

### 5.5 AI 续写/扩写辅助（卡文急救）

**触发方式**：在章节编辑器中，选中已有文本，点击「AI 续写」或「AI 扩写」。

**续写模式**：
- 系统读取：当前章节已有内容 + 剧情节点目标摘要 + 参与角色档案 + 世界观相关设定
- 生成：300~1000 字的续写草稿（提供 3 种不同走向供选择）
- 作者可选择「采用」「修改后采用」「放弃」

**扩写模式**：
- 作者选中一段简略文本（如「主角打败了反派」）
- AI 扩写为详细场景描写（动作、环境、心理、对话）

**爽点节奏模式（V2）**：
- 针对商业化网文，提供「打脸」「升级」「反转」等爽点模板
- AI 根据当前剧情节奏，建议下一章的核心爽点类型

## 九、MVP 范围（第一版可交付）

### 必须包含（P0）

1. **角色小像**：角色 CRUD + 完整档案字段 + 关联角色
2. **剧情分支树**：节点增删改 + 父子连线 + 画布操作 + 伏笔标记
3. **人物关系图**：基础节点增删改 + 关系连线 + 力导向布局
4. **世界观设定库**：设定分类 CRUD + 修炼体系编辑器 + 年表
5. **灵感便签**：快速记录 + 列表 + 拖拽关联到节点/角色
6. **AI 起名助手**：角色名/地名/招式名生成
7. **跨模块跳转**：三大模块间的基础链接跳转
8. **全局搜索**：跨模块搜索

### 暂缓（P1，V1.1 或 V2）

- AI 形象生成（需要文生图 API，成本较高）
- AI 人设一致性校验（需要 LLM 接入，精度需调优）
- 设定规则冲突检测（需要规则引擎，实现复杂）
- AI 续写/扩写（需要 LLM 长文本能力）
- 地图可视化编辑器（Canvas 开发周期长）
- 关系图动态演变（时间轴 Slider）
