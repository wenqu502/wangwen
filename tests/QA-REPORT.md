## 一、致命问题（已修复）

### 🔴 P0: SearchModal.tsx 拼写错误导致白屏崩溃

**问题描述**: `src/components/search/SearchModal.tsx` 第 24、26 行存在拼写错误：
- `useStateStateState<SearchResultItem[]>([])` → 应为 `useState`
- `useRefRefRef<HTMLInputElement>(null)` → 应为 `useRef`

**影响范围**: 应用启动时直接抛出 `ReferenceError: useStateState is not defined`，ErrorBoundary 捕获后显示"出错了"降级页面，整个应用无法使用。

**修复状态**: ✅ 已修复（commit `24fff61`）

**根因分析**: 批量编辑时引入的拼写错误，TypeScript 编译通过但运行时崩溃。后续建议对这类增量编辑引入自动检查。

| `tests/qa-visual.spec.ts` | 1 | 1 | 0 |

**wangwen.spec.ts 失败原因分析**:
- 10 个失败全部是因为测试断言选择器与 DOM 结构不匹配（**测试代码问题，非 App Bug**）
- 例如：`getByRole('heading', { name: '设置' })` 不存在，设置标题是 `<p>` 标签
- 例如：`getByText('角色 1')` 匹配到 2 个元素（列表项 + 详情标题）
- 例如：`getByPlaceholder('sk-...')` 在 API Key 输入框中不存在该 placeholder

**建议**: 重写 `wangwen.spec.ts` 选择器，使用 `exact: true` 和更精确的定位策略。
