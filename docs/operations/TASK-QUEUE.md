## Review 规范

1. **触发条件**: 任何 Agent 完成代码修改后
2. **Review 流程**:
   - 修改 Agent → 提交代码变更
   - Wydq → 通知 `program-system-architecture-expert` 进行 Review
   - 架构师 → 输出 Review 报告（通过/需修改/拒绝）
   - Wydq → 根据 Review 结果更新任务状态
3. **Review 检查项**:
   - 类型安全（tsc -b 通过）
   - 性能影响（渲染次数、内存占用）
   - 安全合规（XSS、CSP、加密）
   - 代码风格一致性
   - 向后兼容性
