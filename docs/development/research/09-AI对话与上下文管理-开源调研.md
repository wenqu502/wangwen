## 一、需求背景与范围界定

### 1.1 PRD 原文需求摘录

根据《PRD-网文创作辅助工具-v3.0.md》第 4 节「全局 AI 创作助手」：

> **能力范围**：
> - 意图理解：理解用户的自然语言创作意图，转化为结构化操作
> - 内容生成：基于用户描述，生成角色、剧情、关系、体系等内容
> - 上下文记忆：记住当前作品的全部设定，确保生成内容前后一致
> - 多轮对话：支持"改一下""再加一个""把它删掉"等上下文延续指令
> - 主动建议：在检测到潜在问题时（如角色崩坏风险、伏笔未回收），主动提醒
>
> **对话上下文管理**：
> AI 助手需要维护以下上下文信息：
> - 当前作品全部角色档案
> - 当前作品全部剧情节点
> - 当前作品全部关系数据
> - 当前作品全部体系设定
> - 当前作品全部灵感便签
> - 用户最近的 20 轮对话历史

性能验收标准：
- AI 对话响应时间 < 5s（常规生成）
- 复杂生成（如剧情树））< 15s
- 首字延迟 < 2s

### 1.2 功能需求拆解

| 编号 | 功能点 | 优先级 | 说明 | 验收标准 |
|------|--------|--------|------|----------|

## 二、AI SDK 技术选型

### 2.1 候选方案概览

#### 方案 A：OpenAI SDK（当前使用）

- **GitHub**：https://github.com/openai/openai-node
- **周下载量**：~880万
- **Bundle**：~34KB gzipped

**特性**：直接调用 API，支持流式输出，通过 baseURL 可对接兼容 OpenAI 格式的第三方 API。

**劣势**：仅支持 OpenAI 格式，切换供应商需手动修改，无内置重试降级。

#### 方案 B：Vercel AI SDK（推荐升级）

- **官网**：https://sdk.vercel.ai/
- **GitHub**：https://github.com/vercel/ai
- **周下载量**：~200万

**特性**：多供应商抽象（25+），React Hook（useChat/useCompletion），结构化输出（Zod Schema），工具调用，自动重试。

```typescript

  baseURL: 'https://api.deepseek.com/v1',
  apiKey: localStorage.getItem('wangwen:api-key'),
})

function ChatPanel() {
  const { messages, input, handleInputChange, handleSubmit, isLoading } = useChat({
    model: deepseek('deepseek-chat'),
    maxTokens: 4096,
  })
  // ...
}
```

#### 方案 C：LangChain.js

- **官网**：https://js.langchain.com/
- **Bundle**：~101KB gzipped

**特性**：最全面的 AI 框架，支持 50+ 供应商，RAG，Agent 工作流。

**劣势**：Bundle 大，学习曲线陡峭，对纯前端项目过度设计。

### 2.2 加权评分矩阵

| 维度 | 权重 | OpenAI SDK | Vercel AI SDK | LangChain.js |
|------|------|------------|---------------|--------------|
| 多供应商 | 25% | 5 | 10 | 9 |
| Bundle 体积 | 15% | 9 | 5 | 3 |

## 三、上下文管理方案

### 3.1 核心问题

全部上下文可能超过 100K tokens（50-100 个角色 × 1000 字 + 100-500 个节点 × 500 字），远超有效使用范围。

### 3.2 分层上下文压缩

```typescript
interface ContextLayer {
  priority: number      // 1-10
  content: string
  tokens: number
  compressible: boolean // 是否可摘要
}

class ContextBuilder {
  private maxTokens = 24000 // 预留 16K 给回复

  async build(workId: string, userMessage: string): Promise<string> {
    const layers: ContextLayer[] = [
      { priority: 1, content: await this.getSystemPrompt(), tokens: 500, compressible: false },
      { priority: 2, content: await this.getRecentChatHistory(workId, 10), tokens: 3000, compressible: true },
      { priority: 3, content: await this.getActivePlotNodes(workId), tokens: 5000, compressible: true },
      { priority: 4, content: await this.getMainCharacters(workId, 10), tokens: 4000, compressible: true },
      { priority: 5, content: await this.getRelevantIdeas(workId), tokens: 2000, compressible: true },
      { priority: 6, content: await this.getSystemRules(workId), tokens: 3000, compressible: true },
      { priority: 7, content: await this.getFullCharactersSummary(workId), tokens: 6000, compressible: true },
    ]

    let totalTokens = 0
    const selected: string[] = []

    for (const layer of layers.sort((a, b) => a.priority - b.priority)) {
      if (totalTokens + layer.tokens <= this.maxTokens) {
        selected.push(layer.content)
        totalTokens += layer.tokens
      } else if (layer.compressible) {
        const compressed = await this.summarizeWithAI(layer.content, this.maxTokens - totalTokens)
        if (compressed) {
          selected.push(`[摘要] ${compressed}`)
          totalTokens += this.estimateTokens(compressed)
        }
      }
    }

    return selected.join('\n\n')
  }

  private estimateTokens(text: string): number {
    const chinese = (text.match(/[\u4e00-\u9fff]/g) || []).length
    const english = (text.match(/[a-zA-Z]+/g) || []).length
    return Math.ceil(chinese + english * 0.75)
  }
}
```

### 3.3 上下文相关性过滤

用户消息"帮我把顾沉的性格改得更偏执"，只需加载「顾沉」的角色档案，无需加载全部角色：

```typescript
async function getRelevantContext(workId: string, userMessage: string): Promise<string> {
  // 1. 提取用户消息中的实体名称
  const characterNames = await extractEntities(userMessage, 'character')
  const plotKeywords = await extractEntities(userMessage, 'plot')

  // 2. 只加载相关实体
  const relevantChars = await db.characters
    .where('workId')
    .equals(workId)
    .filter(c => characterNames.includes(c.name))
    .toArray()

  // 3. 组装上下文
  return formatContext({ characters: relevantChars })
}
```

## 四、结构化输出方案

### 4.1 Zod Schema + Vercel AI SDK

```typescript
import { generateObject } from 'ai'
import { z } from 'zod'

const CharacterSchema = z.object({
  name: z.string().describe('角色姓名'),
  aliases: z.array(z.string()).describe('别名/绰号'),
  tags: z.array(z.string()).describe('标签，如["男主","外科医生"]'),
  appearance: z.string().describe('外貌描写，200字以内'),
  personality: z.object({
    keywords: z.array(z.string()),
    surface: z.string().describe('外在表现'),
    inner: z.string().describe('内心世界'),
    stressResponse: z.string().describe('应激反应'),
  }),
  background: z.string().describe('背景故事'),
  goals: z.string().optional(),
  arc: z.string().optional(),
  quotes: z.array(z.string()).max(3),
})

type CharacterOutput = z.inferfer<typeof CharacterSchema>

async function generateCharacter(description: string): Promise Promise<CharacterOutput> {
  const { object } = await generateObject({
    model: deepseek('deepseek-chat'),
    schema: CharacterSchema,
    prompt: `根据以下描述生成角色档案：\n${description}`,
  })
  return object
}
```

### 4.2 流式结构化输出

对于剧情树生成等长输出，使用流式 JSON：

```typescript
import { streamObject } from 'ai'

const PlotNodeSchema = z.object({
  title: z.string(),
  summary: z.string(),
  status: z.enum(['todo', 'doing', 'done', 'abandoned']),
  parentId: z.string().optional(),
})

async function* generatePlotTree(premise: string) {
  const stream = streamObject({
    model: deepseek('deepseek-chat'),
    schema: z.object({ nodes: z.array(PlotNodeSchema) }),
    prompt: `根据以下梗概生成剧情骨架：\n${premise}`,
  })

  for await (const partial of stream.partialObjectStream) {
    yield partial.nodes || []
  }
}
```

```typescript
  const configs = getEnabledConfigs().sort((a, b) => a.priority - b.priority)
```

## 七、Token 用量监控

```typescript
interface TokenUsage {
  promptTokens: number
  completionTokens: number
  totalTokens: number
  model: string
  timestamp: number
}

class TokenTracker {
  private usages: TokenUsage[] = []

  record(usage: TokenUsage) {
    this.usages.push(usage)
    // 持久化到本地
    db.tokenUsages.add({ ...usage, id: generateId() })
  }

  getDailyUsage(date: string): number {
    return this.usages
      .filter(u => new Date(u.timestamp).toISOString().split('T')[0] === date)
      .reduce((sum, u) => sum + u.totalTokens, 0)
  }

  getEstimatedCost(): number {
    // DeepSeek: 输入 ¥1/M tokens, 输出 ¥2/M tokens
    return this.usages.reduce((cost, u) => {
      return cost + (u.promptTokens / 1e6 * 1) + (u.completionTokens / 1e6 * 2)
    }, 0)
  }
}
```

```typescript
interface ChatBranch {
  id: string
  parentBranchId: string | null
  workId: string
  messages: ChatMessage[]
  createdAt: number
}

class ChatBranchManager {
  async createBranch(parentBranchId: string, fromMessageIndex: number): Promise<string> {
    const parent = await db.chatBranches.get(parentBranchId)
    if (!parent) throw new Error('父分支不存在')

    const newBranch: ChatBranch = {
      id: generateId(),
      parentBranchId,
      workId: parent.workId,
      messages: parent.messages.slice(0, fromMessageIndex + 1),
      createdAt: Date.now(),
    }

    await db.chatBranches.add(newBranch)
    return newBranch.id
  }

  async getBranchTree(workId: string): Promise Promise<BranchNode[]> {
    const branches = await db.chatBranches.where('workId').equals(workId).toArray()
    return buildTree(branches)
  }
}
```

## 九、风险评估

| 风险 | 影响 | 概率 | 应对 |
|------|------|------|------|
| API 服务不可用 | 高 | 低 | 多模型配置，本地降级模式 |
