## 一、需求背景与范围界定

### 1.1 PRD 原文需求摘录

根据《PRD-网文创作辅助工具-v3.0.md》第 5.3 节「人物关系图」：

> **核心理念**：AI 自动分析角色档案和剧情，生成关系网络。
>
> **AI 自动分析生成**：
> - 分析角色档案和剧情节点，推断关系
> - 检测潜在矛盾（如剧情与档案不一致）
> - 生成关系网络图，用户可拖拽调整布局
>
> **AI 对话式修改关系**：
> - 通过自然语言修改关系类型和状态
> - 连锁更新：关系变更后同步更新角色档案、剧情节点标记
> - 支持隐藏关系（虚线显示，默认不对外展示）
>
> **关系类型（可自定义）**：
> - 情感类：爱慕、暗恋、前任、宿敌、挚友、情敌
> - 血缘类：父子、兄妹、师徒
> - 利益类：同盟、上下级、交易、竞争
> - 隐藏类：真实身份、暗中操控、秘密关联

根据《PRD-网文创作辅助工具-v2.0.md》第 4.3 节补充交互需求：

> - **点击角色节点**：展开角色小像侧边栏，快速查看档案
> - **点击关系线**：显示关系详情（建立时间、关键事件、当前状态）
> - **圈层布局**：以核心角色为中心，按亲疏远近自动分层
> - **势力着色**：同一阵营的角色节点颜色统一
> - **动态演变**：按「卷/章节」滑动时间轴，观看关系网络随剧情演变（角色死亡后节点变灰，关系线变虚）
> - **新增关系**：拖拽一个角色到另一个角色上，弹出关系创建弹窗
> - **关系连线样式**：
>   - 血缘：实线 + 暖色（橙/红）
>   - 情感：实线 + 图标（心形/闪电）
>   - 利益：虚线 + 冷色（蓝/灰）
>   - 冲突：粗线 + 红色
>   - 隐藏：点线 + 灰色（可切换显隐）

## 三、候选项目深度分析

### 3.1 AntV G6 v5 —— 阿里开源图可视化引擎

| 项 | 内容 |
|----|------|
| 官网 | https://g6.antv.antgroup.com |
| GitHub | https://github.com/antvis/G6 |
| Stars | 12k+ |
| 最新版本 | v5.0（2024 年发布，全面重构） |
| 技术栈 | TypeScript + Canvas 2D / SVG（双渲染器） |
| 开源协议 | MIT |
| 定位 | 专业级图可视化与分析引擎 |
| 维护团队 | 阿里 AntV 团队 |
| 更新频率 | 每月 2-4 次发版 |

**核心能力详解**

1. **双渲染器架构**：v5 支持 Canvas 2D 和 SVG 两种渲染后端，可根据场景切换。Canvas 适合大规模图（性能优先），SVG 适合需要 CSS 样式控制的场景。

2. **布局算法库**：内置 10+ 种布局算法：
   - `force`：经典力导向布局，支持自定义引力和斥力
   - `fruchterman`：Fruchterman-Reingold 力导向
   - `force-atlas2`：ForceAtlas2（更适合社交网络）
   - `circular`：环形布局
   - `radial`：辐射状布局（适合圈层布局需求）
   - `concentric`：同心圆布局（适合势力分层）
   - `grid`：网格布局
   - `dagre`：层次布局（不适合无环图但可备用）

3. **自定义节点（G6 v5 新特性）**：
   - 支持用 React 组件定义节点（`@antv/g6-react-node`）
   - 支持 JSX 语法定义节点图形
   - 节点可组合基础图形（rect、circle、text、image、path 等）

4. **自定义边**：
   - 支持实线、虚线、点线（`lineDash` 属性）
   - 支持曲线（quadratic、cubic、arc）
   - 支持边标签（label）自动避让
   - 支持箭头自定义

5. **交互系统**：
   - 内置拖拽（drag-node）、缩放（zoom-canvas）、平移（drag-canvas）
   - 框选（brush-select）、lasso 选择
   - 右键菜单（contextmenu）插件
   - Hover 高亮（hover-activate）
   - 点击选中（click-select）

6. **插件生态**：
   - `minimap`：小地图导航
   - `grid`：网格背景
   - `toolbar`：工具栏（缩放/适应屏幕/重置）
   - `tooltip`：悬浮提示
   - `menu`：右键菜单
   - `legend`：图例
   - `snapshot`：导出图片

**与织文需求的匹配度分析**

| 织文需求 | G6 支持情况 | 实现复杂度 |
|----------|-------------|------------|
| 力导向布局 | 内置 force/fruchterman/force-atlas2 | 低 |
| 角色卡片节点 | React Node 或自定义图形组合 | 中 |
| 关系连线样式 | 完全支持（线型/颜色/粗细/箭头） | 低 |
| 关系标签 | 内置 label 配置，支持自动避让 | 低 |
| 拖拽调整布局 | 内置 drag-node 交互 | 低 |
| 拖拽创建关系 | 需自定义交互（drag-node + 检测落点） | 中 |
| 隐藏关系开关 | 通过数据过滤 + 重新渲染实现 | 低 |
| 势力着色 | 节点样式动态绑定阵营数据 | 低 |
| 圈层布局 | 内置 radial/concentric 布局 | 低 |
| 动态演变 | 数据驱动 + 时间轴控制 + 过渡动画 | 中 |
| 右键菜单 | 内置 menu 插件 | 低 |
| 画布缩放平移 | 内置 | 低 |
| 小地图 | 内置 minimap 插件 | 低 |
| 导出 PNG | 内置 snapshot 插件 | 低 |

**优缺点**

| 优点 | 缺点 |
|------|------|
| 中文文档完善，国内社区活跃 | v5 相对较新，部分插件生态仍在完善 |
| 布局算法丰富，力导向效果专业 | 包体积较大（~500KB gzip） |
| 双渲染器（Canvas/SVG）灵活选择 | 学习曲线中等，概念较多 |
| React 节点支持，与项目技术栈匹配 | 自定义 React 节点时性能需谨慎优化 |
| 插件生态完善，常用功能开箱即用 | 超大规模图（1000+ 节点）性能不如纯 WebGL 方案 |
| MIT 协议，可商用 | — |

**对织文的借鉴点**

| 借鉴项 | 具体应用 |
|--------|----------|
| 力导向布局参数调优 | 调整引力/斥力使关系紧密角色自然靠近 |
| radial 布局 | 实现「以核心角色为中心」的圈层布局需求 |
| concentric 布局 | 实现「按亲疏远近分层」的势力圈层 |
| React Node | 用角色卡片组件作为节点，含头像+名字+标签 |
| 边样式系统 | 不同关系类型映射到不同线型/颜色/粗细 |
| 边标签避让 | 关系类型文字自动避让，避免重叠 |
| menu 插件 | 角色右键菜单（查看详情/编辑关系/删除） |
| minimap 插件 | 大型关系图的缩略导航 |
| snapshot 插件 | 一键导出关系图为 PNG |
| hover-activate | 悬停节点高亮相关连线 |

### 3.2 D3.js + d3-force + d3-selection + d3-zoom —— 底层数据可视化库

| 项 | 内容 |
|----|------|
| 官网 | https://d3js.org |
| GitHub | https://github.com/d3/d3 |
| Stars | 110k+ |
| 最新版本 | v7.x |
| 技术栈 | JavaScript + SVG / Canvas / WebGL |
| 开源协议 | ISC |
| 定位 | 底层数据可视化库，提供数据到图形的映射能力 |
| 维护团队 | Observable / Mike Bostock |
| 更新频率 | 稳定维护，年度大版本 |

**核心能力详解**

1. **d3-force**：物理仿真引擎
   - `forceManyBody`：节点间斥力（类似电荷）
   - `forceLink`：连线引力（弹簧模型）
   - `forceCenter`：向画布中心引力
   - `forceCollide`：节点碰撞检测（防止重叠）
   - `forceX/forceY`：定向引力
   - `forceRadial`：径向布局力（适合圈层布局）
   - 完全可自定义力参数和组合

2. **d3-selection**：数据绑定与 DOM 操作
   - `data().join()` 模式实现数据驱动更新
   - 事件监听（click、drag、zoom）
   - 过渡动画（transition）

3. **d3-zoom**：缩放与平移
   - 滚轮缩放、拖拽平移
   - 缩放范围限制、过渡动画
   - 支持 SVG 和 Canvas

4. **d3-drag**：拖拽交互
   - 节点拖拽
   - 拖拽开始/进行中/结束的事件回调

**与织文需求的匹配度分析**

| 织文需求 | D3 支持情况 | 实现复杂度 |
|----------|-------------|------------|
| 力导向布局 | d3-force 完全支持，参数可调 | 中 |
| 角色卡片节点 | 需用 SVG foreignObject 或手动绘制 | 高 |
| 关系连线样式 | SVG path 完全可控 | 中 |
| 关系标签 | 手动计算位置，需自行实现避让 | 高 |
| 拖拽调整布局 | d3-drag 支持 | 中 |
| 拖拽创建关系 | 需自行实现拖拽检测逻辑 | 高 |
| 隐藏关系开关 | 数据过滤 + 重新绑定 | 中 |
| 势力着色 | 数据驱动样式绑定 | 低 |
| 圈层布局 | forceRadial 或自定义力 | 中 |
| 动态演变 | 数据驱动更新 + transition | 中 |
| 右键菜单 | 需自行实现 | 高 |
| 画布缩放平移 | d3-zoom 支持 | 中 |
| 小地图 | 需自行实现 | 高 |
| 导出 PNG | Canvas 方案或 html2canvas | 中 |

**优缺点**

| 优点 | 缺点 |
|------|------|
| 灵活性极高，可做任何自定义效果 | 学习曲线极其陡峭 |
| 社区最庞大，资料最丰富 | 需要手写大量样板代码 |
| 性能可控（可选择 SVG/Canvas/WebGL） | 不自带任何 UI 组件 |
| 包体积极小（按需引入模块） | 开发周期显著增长 |
| 力导向仿真参数完全可控 | 关系标签避让需自行实现 |
| 适合作为底层引擎封装 | React 集成不如专用库自然 |

**对织文的借鉴点**

| 借鉴项 | 具体应用 |
|--------|----------|
| d3-force 参数调优 | 自定义斥力/引力使关系图更美观 |
| forceRadial | 实现「以核心角色为中心」的径向布局 |
| forceCollide | 防止角色节点重叠 |
| transition | 关系动态演变时的平滑过渡动画 |
| zoom transform | 画布缩放状态的精确控制 |

### 3.3 React Force Graph —— React 力导向图组件

| 项 | 内容 |
|----|------|
| GitHub | https://github.com/vasturiano/react-force-graph |
| Stars | 5k+ |
| 最新版本 | v1.x |
| 技术栈 | React + Three.js（3D）/ Canvas 2D（2D） |
| 开源协议 | MIT |
| 定位 | React 封装的力导向图组件 |
| 维护团队 | Vasco Asturiano（个人开发者） |
| 更新频率 | 持续维护 |

**核心能力详解**

1. **三种渲染模式**：
   - `ForceGraph2D`：Canvas 2D 渲染，适合关系图
   - `ForceGraph3D`：Three.js WebGL 渲染，3D 力导向图
   - `ForceGraphVR`：VR 场景（织文不需要）

2. **内置力导向引擎**：
   - 基于 d3-force 的力导向仿真
   - 支持自定义力参数（引力、斥力、距离）
   - 支持暂停/恢复仿真

3. **React 组件化 API**：
   - 完全作为 React 组件使用
   - props 驱动配置（节点数据、连线数据、颜色、大小等）
   - 回调函数处理交互（onNodeClick、onLinkClick、onNodeDrag 等）

4. **节点/边自定义**：
   - `nodeCanvasObject`：Canvas 绘制自定义节点
   - `nodeThreeObject`：Three.js 自定义节点（3D）
   - `linkCanvasObject`：Canvas 绘制自定义边
   - `nodeLabel`/`linkLabel`：悬浮标签

5. **交互内置**：
   - 拖拽节点（可固定位置）
   - 滚轮缩放
   - 悬停高亮
   - 点击回调

**与织文需求的匹配度分析**

| 织文需求 | React Force Graph 支持情况 | 实现复杂度 |
|----------|---------------------------|------------|
| 力导向布局 | 内置，基于 d3-force | 低 |
| 角色卡片节点 | nodeCanvasObject 自定义绘制 | 中 |
| 关系连线样式 | linkCanvasObject 自定义 | 中 |
| 关系标签 | linkLabel 或自定义绘制 | 中 |
| 拖拽调整布局 | 内置，支持固定位置 | 低 |
| 拖拽创建关系 | 需自行实现（检测拖拽落点） | 高 |
| 隐藏关系开关 | 数据过滤后重新传入 props | 低 |
| 势力着色 | nodeColor 回调 | 低 |
| 圈层布局 | 不支持，纯力导向 | 高 |
| 动态演变 | 数据驱动 + 过渡需自行实现 | 中 |
| 右键菜单 | 需自行实现 | 高 |
| 画布缩放平移 | 内置 | 低 |
| 小地图 | 不支持 | 高 |
| 导出 PNG | Canvas 原生 toDataURL | 低 |

**优缺点**

| 优点 | 缺点 |
|------|------|
| React 原生组件，接入简单 | 自定义能力有限（Canvas 绘制不如 DOM 灵活） |
| API 简洁，props 驱动 | 不支持圈层布局等高级布局 |
| 2D/3D 切换方便 | 大规模图（100+ 节点）性能下降明显 |
| 内置交互丰富 | 无右键菜单、小地图等插件 |
| Canvas 导出方便 | 社区相对较小 |

**对织文的借鉴点**

| 借鉴项 | 具体应用 |
|--------|----------|
| nodeCanvasObject | Canvas 绘制角色头像圆形裁剪 |
| linkCanvasObject | Canvas 绘制带标签的关系连线 |
| 固定节点位置 | 用户拖拽后固定，不再受力导向影响 |
| props 驱动数据更新 | React 风格的数据流 |

## 四、综合评分与选型决策

### 4.1 评分矩阵

| 维度（权重） | AntV G6 v5 | D3.js | React Force Graph | Cytoscape.js | Sigma.js |
|-------------|-----------|-------|------------------|-------------|----------|
| 技术栈匹配度（25%） | 5 | 3 | 5 | 3 | 3 |
| 自定义能力（20%） | 4 | 5 | 3 | 4 | 5 |
| 内置功能完备度（20%） | 5 | 2 | 3 | 4 | 2 |
| 性能表现（15%） | 4 | 4 | 3 | 4 | 5 |
| 社区活跃度（10%） | 4 | 5 | 3 | 4 | 3 |
| 许可证（10%） | 5 | 5 | 5 | 5 | 5 |
| **加权总分** | **4.60** | **3.55** | **3.65** | **3.85** | **3.45** |

**评分细则**

- **AntV G6 v5（4.60/5）**：
  - 技术栈匹配度 5：原生 TypeScript，提供 React Node 支持
  - 自定义能力 4：React 节点 + 自定义图形，能满足角色卡片需求
  - 内置功能完备度 5：布局、交互、插件生态最完整
  - 性能表现 4：Canvas 渲染，50 节点场景流畅
  - 社区活跃度 4：阿里团队维护，中文社区活跃

- **D3.js（3.55/5）**：
  - 自定义能力 5、社区活跃度 5 为优势
  - 内置功能完备度 2 为致命短板（无内置组件）
  - 适合作为底层引擎，不适合直接用于项目开发

- **React Force Graph（3.65/5）**：
  - 技术栈匹配度 5、接入简单为优势
  - 自定义能力 3、不支持圈层布局为短板
  - 适合快速原型，不适合复杂定制需求

- **Cytoscape.js（3.85/5）**：
  - 网络图领域专业，扩展生态丰富
  - React 集成较弱，样式系统学习成本
  - 可作为备选方案

- **Sigma.js（3.45/5）**：
  - WebGL 性能最优，但 50 节点场景优势不明显
  - 学习曲线陡峭，开发成本高
  - 适合未来大规模图场景预留

### 4.2 选型结论

**主选方案：AntV G6 v5**

理由：
1. **综合得分最高**（4.60/5），在六个维度上均衡优秀
2. **功能完备度最高**：力导向布局、圈层布局、拖拽、缩放、小地图、右键菜单、导出全部内置或插件支持
3. **技术栈最匹配**：原生 TypeScript，支持 React 组件渲染节点，与项目 React 18 + TypeScript 完美契合
4. **开发效率最高**：内置功能覆盖 80% 需求，开发周期最短
5. **中文生态**：文档、社区、示例均为中文，团队上手快

**备选方案：Cytoscape.js**

若 G6 v5 在以下场景不满足时切换：
- G6 的 React Node 性能在 50+ 节点场景出现瓶颈
- 需要更专业的网络图分析功能（如最短路径、社区发现）

**预留方案：Sigma.js + graphology**

若未来作品角色数量超过 200+，需要 WebGL 高性能渲染时评估迁移。

```typescript
import { persist } from 'zustand/middleware'
import {
    Character,
    Relationship,
    RelationType,

    RelationCategory,
    RelationStatus,

interface RelationshipGraphStore extends RelationshipGraphState {
    // 角色操作
    addCharacter: (character: Character) => void
    updateCharacter: (id: string, updates: Partial Partial<Character>) => void
    removeCharacter: (id: string) => void
    setCharacterPosition: (id: string, x: number, y: number) => void

    // 关系操作
    addRelationship: (relationship: Relationship) => void
    updateRelationship: (id: string, updates: Partial Partial<Relationship>) => void
    removeRelationship: (id: string) => void

    // 关系类型操作
    addRelationType: (type: RelationType) => void
    removeRelationType: (id: string) => void

    // 视图控制
    toggleHiddenRelations: () => void
    selectCharacter: (id: string | null) => void
    selectRelationship: (id: string | null) => void
    setLayoutType: (type: 'force' | 'radial' | 'concentric') => void
    setZoom: (zoom: number) => void
    setCenter: (center: [number, number]) => void

    // 时间轴
    setCurrentChapter: (chapter: number) => void

    // AI 生成结果写入
    setCharacters: (characters: Character[]) => void
    setRelationships: (relationships: Relationship[]) => void

    // 导入导出
    exportToJSON: () => string
    importFromJSON: (json: string) => void
}

export const useRelationshipGraphStore = create create<RelationshipGraphStore>()(
    immer(
        persist(
            (set, get) => ({
                // 初始状态
                characters: [],
                relationships: [],
                relationTypes: [
                    { id: 'love', name: '爱慕', category: RelationCategory.EMOTION },
                    { id: 'secret-love', name: '暗恋', category: RelationCategory.EMOTION },
                    { id: 'ex', name: '前任', category: RelationCategory.EMOTION },
                    { id: 'rival', name: '宿敌', category: RelationCategory.EMOTION },
                    { id: 'friend', name: '挚友', category: RelationCategory.EMOTION },
                    { id: 'love-rival', name: '情敌', category: RelationCategory.EMOTION },
                    { id: 'father-son', name: '父子', category: RelationCategory.BLOOD },
                    { id: 'sibling', name: '兄妹', category: RelationCategory.BLOOD },
                    { id: 'master-apprentice', name: '师徒', category: RelationCategory.BLOOD },
                    { id: 'ally', name: '同盟', category: RelationCategory.INTEREST },
                    { id: 'superior', name: '上下级', category: RelationCategory.INTEREST },
                    { id: 'trade', name: '交易', category: RelationCategory.INTEREST },
                    { id: 'compete', name: '竞争', category: RelationCategory.INTEREST },
                    { id: 'hidden-identity', name: '真实身份', category: RelationCategory.HIDDEN },
                    { id: 'manipulate', name: '暗中操控', category: RelationCategory.HIDDEN },
                    { id: 'secret-link', name: '秘密关联', category: RelationCategory.HIDDEN },
                ],
                factions: [],
                timeline: [],
                currentChapter: 0,
                showHiddenRelations: true,
                selectedCharacterId: null,
                selectedRelationshipId: null,
                layoutType: 'force',
                zoom: 1,
                center: [0, 0],

                // 角色操作
                addCharacter: (character) =>
                    set((state) => {
                        state.characters.push(character)
                    }),

                updateCharacter: (id, updates) =>
                    set((state) => {
                        const index = state.characters.findIndex((c) => c.id === id)
                        if (index !== -1) {
                            Object.assign(state.characters[index], updates)
                        }
                    }),

                removeCharacter: (id) =>
                    set((state) => {
                        state.characters = state.characters.filter((c) => c.id !== id)
                        // 级联删除相关关系
                        state.relationships = state.relationships.filter(
                            (r) => r.source !== id && r.target !== id
                        )
                    }),

                setCharacterPosition: (id, x, y) =>
                    set((state) => {
                        const char = state.characters.find((c) => c.id === id)
                        if (char) {
                            char.x = x
                            char.y = y
                        }
                    }),

                // 关系操作
                addRelationship: (relationship) =>
                    set((state) => {
                        state.relationships.push(relationship)
                    }),

                updateRelationship: (id, updates) =>
                    set((state) => {
                        const index = state.relationships.findIndex((r) => r.id === id)
                        if (index !== -1) {
                            Object.assign(state.relationships[index], updates)
                        }
                    }),

                removeRelationship: (id) =>
                    set((state) => {
                        state.relationships = state.relationships.filter((r) => r.id !== id)
                    }),

                // 关系类型操作
                addRelationType: (type) =>
                    set((state) => {
                        state.relationTypes.push(type)
                    }),

                removeRelationType: (id) =>
                    set((state) => {
                        state.relationTypes = state.relationTypes.filter((t) => t.id !== id)
                    }),

                // 视图控制
                toggleHiddenRelations: () =>
                    set((state) => {
                        state.showHiddenRelations = !state.showHiddenRelations
                    }),

                selectCharacter: (id) =>
                    set((state) => {
                        state.selectedCharacterId = id
                        state.selectedRelationshipId = null
                    }),

                selectRelationship: (id) =>
                    set((state) => {
                        state.selectedRelationshipId = id
                        state.selectedCharacterId = null
                    }),

                setLayoutType: (type) =>
                    set((state) => {
                        state.layoutType = type
                    }),

                setZoom: (zoom) =>
                    set((state) => {
                        state.zoom = Math.max(0.25, Math.min(2, zoom))
                    }),

                setCenter: (center) =>
                    set((state) => {
                        state.center = center
                    }),

                // 时间轴
                setCurrentChapter: (chapter) =>
                    set((state) => {
                        state.currentChapter = chapter
                    }),

                addTimelineSnapshot: (snapshot) =>
                    set((state) => {
                        state.timeline.push(snapshot)
                        state.timeline.sort((a, b) => a.chapter - b.chapter)
                    }),

                // AI 批量写入
                setCharacters: (characters) =>
                    set((state) => {
                        state.characters = characters
                    }),

                setRelationships: (relationships) =>
                    set((state) => {
                        state.relationships = relationships
                    }),

                // 导入导出
                exportToJSON: () => {
                    const { characters, relationships, relationTypes, factions, timeline } = get()
                    return JSON.stringify(
                        { characters, relationships, relationTypes, factions, timeline },
                        null,
                        2
                    )
                },

                importFromJSON: (json) =>
                    set((state) => {
                        try {
                            const data = JSON.parse(json)
                            state.characters = data.characters || []
                            state.relationships = data.relationships || []
                            state.relationTypes = data.relationTypes || state.relationTypes
                            state.factions = data.factions || []
                            state.timeline = data.timeline || []
                        } catch (e) {
                            console.error('Import failed:', e)
                        }
```

### 7.2 关系图画布组件

```tsx
// src/modules/relationship-graph/components/RelationshipCanvas.tsx

import React, { useEffect, useRef, useCallback } from 'react'
import { Graph } from '@antv/g6'
import { CharacterNode } from './CharacterNode'
import { useRelationshipGraphStore } from '../store'
import { GraphData, G6Node, G6Edge } from '../types'

export const RelationshipCanvas: React.FC = () => {
    const containerRef = useRefRef<HTMLDivElement>(null)
    const graphRef = useRefRef<Graph | null>(null)

    const {
        characters,
        relationships,
        relationTypes,
        showHiddenRelations,
        selectedCharacterId,
        layoutType,
        setCharacterPosition,
        selectCharacter,
        selectRelationship,
    } = useRelationshipGraphStore()

    /** 数据转换：Store 数据 → G6 GraphData */
    const convertToG6Data = useCallback((): GraphData => {
        const nodes: G6Node[] = characters.map((char) => ({
            id: char.id,
            data: char,
            style: {
                x: char.x,
                y: char.y,
            },
        }))

        const edges: G6Edge[] = relationships
            .filter((rel) => showHiddenRelations || rel.category !== 'hidden')
            .map((rel) => ({
                id: rel.id,
                source: rel.source,
                target: rel.target,
                data: rel,
            }))

        return { nodes, edges }
    }, [characters, relationships, showHiddenRelations])

    /** 获取关系类型样式配置 */
    const getRelationStyle = useCallback(
        (category: string, status: string, strength: number) => {
            const baseWidth = Math.max(1, strength / 3)

            switch (category) {
                case 'emotion':
                    return {
                        stroke: '#ec4899',
                        lineWidth: baseWidth,
                        lineDash: status === 'former' ? [4, 4] : [],
                    }
                case 'blood':
                    return {
                        stroke: '#f97316',
                        lineWidth: baseWidth,
                        lineDash: [],
                    }
                case 'interest':
                    return {
                        stroke: '#3b82f6',
                        lineWidth: baseWidth,
                        lineDash: [6, 3],
                    }
                case 'conflict':
                    return {
                        stroke: '#dc2626',
                        lineWidth: baseWidth + 1,
                        lineDash: [],
                    }
                case 'hidden':
                    return {
                        stroke: '#6b7280',
                        lineWidth: baseWidth,
                        lineDash: [2, 4],
                        opacity: 0.6,
                    }
                default:
                    return {
                        stroke: '#9ca3af',
                        lineWidth: baseWidth,
                    }
            }
        },
        []
    )

    /** 初始化 G6 图实例 */
    useEffect(() => {
        if (!containerRef.current || graphRef.current) return

        const graph = new Graph({
            container: containerRef.current,
            width: containerRef.current.clientWidth,
            height: containerRef.current.clientHeight,
            data: convertToG6Data(),
            node: {
                type: 'react',
                component: CharacterNode,
                size: [100, 110],
            },
            edge: {
                style: {
                    endArrow: true,
                    labelBackground: true,
                    labelBackgroundFill: '#ffffff',
                    labelBackgroundOpacity: 0.9,
                    labelBackgroundPadding: [2, 4],
                    labelBackgroundRadius: 4,
                },
            },
            layout: {
                type: layoutType,
                preventOverlap: true,
                nodeSpacing: 30,
            },
            behaviors: [
                'drag-canvas',
                'zoom-canvas',
                'drag-node',
                'click-select',
                {
                    type: 'hover-activate',
                    degree: 1,
                },
            ],
            plugins: [
                {
                    type: 'minimap',
                    size: [140, 90],
                    position: 'bottom-left',
                },
                {
                    type: 'grid',
                    size: 20,
                    stroke: '#f0f0f0',
                },
            ],
        })

        // 节点点击事件
        graph.on('node:click', (evt) => {
            const nodeId = evt.target.id
            selectCharacter(nodeId)
        })

        // 边点击事件
        graph.on('edge:click', (evt) => {
            const edgeId = evt.target.id
            selectRelationship(edgeId)
        })

        // 节点拖拽结束事件（固定位置）
        graph.on('node:dragend', (evt) => {
            const nodeId = evt.target.id
            const model = graph.getNodeData(nodeId)
            if (model?.style?.x !== undefined && model?.style?.y !== undefined) {
                setCharacterPosition(nodeId, model.style.x, model.style.y)
            }
        })

        graphRef.current = graph

        return () => {
            graph.destroy()
            graphRef.current = null
        }
    }, [])

    /** 数据变化时更新图 */
    useEffect(() => {
        if (!graphRef.current) return

        const data = convertToG6Data()
        graphRef.current.setData(data)

        // 更新边样式
        relationships.forEach((rel) => {
            if (!showHiddenRelations && rel.category === 'hidden') return
            const style = getRelationStyle(rel.category, rel.status, rel.strength)
            graphRef.current?.updateEdgeData([
                {
                    id: rel.id,
                    style: {
                        ...style,
                        labelText: rel.typeName,
                        labelFill: style.stroke,
                    },
                },
            ])
        })

        graphRef.current.render()
    }, [characters, relationships, showHiddenRelations, convertToG6Data, getRelationStyle])

    /** 布局变化时重新布局 */
    useEffect(() => {
        if (!graphRef.current) return
        graphRef.current.setLayout({
            type: layoutType,
            preventOverlap: true,
            nodeSpacing: 30,
        })
        graphRef.current.layout()
    }, [layoutType])

    /** 选中态高亮 */
    useEffect(() => {
        if (!graphRef.current) return

        const states = selectedCharacterId
            ? [{ ids: selectedCharacterId, state: 'selected' }]
            : []

        graphRef.current.setElementState(states)
    }, [selectedCharacterId])

    /** 窗口大小变化自适应 */
    useEffect(() => {
        const handleResize = () => {
            if (containerRef.current && graphRef.current) {
                graphRef.current.setSize(
                    containerRef.current.clientWidth,
                    containerRef.current.clientHeight
                )
            }
        }

        window.addEventListener('resize', handleResize)
        return () => window.removeEventListener('resize', handleResize)
    }, [])

    return (
        <div
            ref={containerRef}
            style={{
                width: '100%',
                height: '100%',
                background: '#fafafa',
                borderRadius: 8,
                overflow: 'hidden',
            }}
        />
    )
}
```

```tsx
// src/modules/relationship-graph/components/CreateRelationModal.tsx

import React, { useState } from 'react'
import { useRelationshipGraphStore } from '../store'
import { RelationCategory, RelationStatus } from '../types'

interface CreateRelationModalProps {
    sourceId: string
    targetId: string
    onClose: () => void
}

export const CreateRelationModal: React.FCC<CreateRelationModalProps> = ({
    sourceId,
    targetId,
    onClose,
}) => {
    const { characters, relationTypes, addRelationship } = useRelationshipGraphStore()
    const [typeId, setTypeId] = useState('')
    const [description, setDescription] = useState('')
    const [strength, setStrength] = useState(5)

    const sourceChar = characters.find((c) => c.id === sourceId)
    const targetChar = characters.find((c) => c.id === targetId)

    const handleSubmit = () => {
        const selectedType = relationTypes.find((t) => t.id === typeId)
        if (!selectedType) return

        addRelationship({
            id: `rel-${Date.now()}`,
            source: sourceId,
            target: targetId,
            typeId: selectedType.id,
            typeName: selectedType.name,
            category: selectedType.category,
            status: selectedType.category === 'hidden' ? RelationStatus.SECRET : RelationStatus.ACTIVE,
            description,
            strength,
            isBidirectional: false,
        })

        onClose()
    }

    return (
        <div className="modal-overlay" onClick={onClose}>
            <div className="modal-content" onClick={(e) => e.stopPropagation()}>
                <h3>创建关系</h3>
                <p>
                    {sourceChar?.name} → {targetChar?.name}
                </p>

                <div className="form-group">
                    <label>关系类型</label>
                    <select value={typeId} onChange={(e) => setTypeId(e.target.value)}>

                        value={strength}

                    />
                </div>

                <div className="modal-actions">
                    <button className="btn-default" onClick={onClose}>
                        取消
                    </button>
                    <button className="btn-primary" onClick={handleSubmit} disabled={!typeId}>
                        确认创建
                    </button>
                </div>
            </div>
        </div>
    )
}
```

## 九、性能优化策略

| 优化点 | 策略 | 预期效果 |
|--------|------|----------|
| 节点渲染 | G6 Canvas 渲染 + 节点层级优化 | 50 节点流畅，100 节点可接受 |
| 标签渲染 | 缩放 < 0.5x 时隐藏标签，避免重叠 | 减少绘制开销 30% |
| 布局计算 | 首次用 force 布局，后续增量更新 | 布局计算 < 2s |
| 数据更新 | 使用 G6 `setData` 局部更新，避免全量重绘 | 更新延迟 < 100ms |
| 防抖处理 | 时间轴滑动使用 200ms debounce | 避免频繁重布局 |
| 关系过滤 | 隐藏关系通过数据过滤实现，不销毁元素 | 切换显隐 < 50ms |
| 图片加载 | 头像使用懒加载 + 占位渐变 | 首屏渲染速度提升 |
| 持久化 | IndexedDB 分片存储（角色/关系/时间轴分离） | 存储/读取 < 500ms |

## 十四、AI 提示词设计（ai-prompts.ts）

```typescript
// src/modules/relationship-graph/ai-prompts.ts

/** AI 分析角色关系提示词 */
export const analyzeRelationshipsPrompt = (characters: string, plotNodes: string) => `
你是一位专业的小说编辑，擅长分析角色关系网络。

请根据以下角色档案和剧情节点，分析并推断角色之间的关系。

## 角色档案
${characters}

## 剧情节点
${plotNodes}

## 分析要求
1. 推断所有显性关系（角色之间直接互动产生的关系）
2. 推断潜在的隐性关系（血缘、隐藏身份、暗中操控等）
3. 检测剧情与角色档案之间的矛盾
4. 为每组关系标注：类型、强度（1-10）、建立时间、关键事件

## 输出格式
返回 JSON 数组，每个元素包含：
- source: 源角色名
- target: 目标角色名
- type: 关系类型（从预设中选择或新增）
- category: 大类（emotion/blood/interest/conflict/hidden）
- strength: 强度 1-10
- description: 关系描述
- establishedAt: 建立时间（章节号）
- keyEvent: 关键事件摘要

同时返回一个 "conflicts" 数组，包含检测到的矛盾。
`

/** AI 修改关系提示词 */
export const modifyRelationshipPrompt = (
    relationship: string,
    modification: string
) => `
用户想要修改以下关系：

当前关系：${relationship}

用户指令：${modification}

请：
1. 更新关系数据
2. 分析此修改对其他角色档案和剧情节点的连锁影响
3. 列出需要复核的剧情节点
4. 如果需要，生成补充设定建议

返回更新后的关系 JSON 和连锁影响分析。
`

关系：${relationship}
上下文：${context}

要求：
- 与现有剧情不矛盾
- 增强人物塑造
- 为后续剧情埋下伏笔
`
```
