> 适用项目：96号项目 — 织文 WangWen
> 调研模块：人物关系图（Relationship Graph）
> 调研日期：2026-05-04

---

## 一、调研背景

织文 WangWen 需要可视化展示作品中的角色关系网络。AI 自动分析角色档案和剧情节点后，生成人物关系图，展示角色之间的情感、血缘、利益、隐藏关系等。用户需要：

- 查看角色关系网络（力导向布局）
- 区分公开关系与隐藏关系（虚线/透明度）
- 点击关系查看详情和编辑
- 拖拽角色调整布局
- 支持右键菜单操作

## 二、候选项目分析

### 2.1 AntV G6 —— 阿里开源图可视化引擎

| 项 | 内容 |
|----|------|
| 官网 | https://g6.antv.antgroup.com |
| GitHub | https://github.com/antvis/G6 |
| Stars | 12k+ |
| 技术栈 | TypeScript + Canvas/SVG |
| 开源协议 | MIT |
| 定位 | 专业级图可视化与分析引擎 |

**核心能力**

- **力导向布局**：内置 Fruchterman、Force、ForceAtlas2 等布局算法
- **自定义节点/边**：完全自定义节点样式（卡片/圆形/图片等）和边样式（实线/虚线/曲线）
- **交互丰富**：拖拽、缩放、框选、右键菜单、Tooltip
- **性能优化**：支持大规模图（1000+ 节点）的流畅渲染
- **插件生态**：Minimap、Grid、Menu、Tooltip 等插件

**对织文的借鉴点**

| 借鉴项 | 具体应用 |
|--------|----------|
| 力导向布局 | 角色关系网络的自动布局，关系越紧密距离越近 |
| 自定义节点 | 用角色卡片作为节点（头像+名字+标签） |
| 自定义边 | 公开关系实线、隐藏关系虚线+半透明、情感关系带颜色 |
| 边标签 | 连线中部显示关系类型（爱慕/疏离/挚友/隐藏关联） |
| 右键菜单插件 | 角色右键：查看详情/编辑关系/添加关联/删除 |
| Minimap 插件 | 大型关系图的缩略导航 |

### 2.2 D3.js —— 数据驱动文档

| 项 | 内容 |
|----|------|
| 官网 | https://d3js.org |
| GitHub | https://github.com/d3/d3 |
| Stars | 110k+ |
| 技术栈 | JavaScript + SVG/Canvas/WebGL |
| 开源协议 | ISC |
| 定位 | 底层数据可视化库 |

**核心能力**

- **完全自由**：从底层控制每一个像素，可做任何自定义效果
- **力导向仿真**：`d3-force` 模块提供物理仿真引擎
- **数据绑定**：数据驱动 DOM 更新
- **过渡动画**：丰富的过渡和插值能力

**优缺点**

| 优点 | 缺点 |
|------|------|
| 灵活性极高 | 学习曲线陡峭 |
| 社区庞大 | 需要手写大量样板代码 |
| 可做任何效果 | 不自带 UI 组件（如 Minimap、右键菜单） |

**对织文的借鉴点**

D3.js 更适合作为底层引擎，若选择 D3，需要自行封装大量交互逻辑。G6 在此基础上做了更高层次的封装。

---

### 2.3 React Force Graph —— React 封装库

| 项 | 内容 |
|----|------|
| GitHub | https://github.com/vasturiano/react-force-graph |
| Stars | 5k+ |
| 技术栈 | React + Three.js / Canvas 2D |
| 开源协议 | MIT |
| 定位 | React 力导向图组件 |

**核心能力**

- **React 组件化**：直接作为 React 组件使用
- **3D/2D 支持**：支持 3D 力导向图（Three.js）和 2D（Canvas）
- **交互内置**：拖拽、缩放、悬停、点击回调
- **节点/边自定义**：通过回调函数返回 JSX/Canvas 绘制指令

**优缺点**

| 优点 | 缺点 |
|------|------|
| React 原生 | 自定义能力不如 G6/D3 |
| 接入简单 | 大规模图性能不如 G6 |
| 3D 效果酷炫 | 织文不需要 3D |

## 三、技术选型建议

### 推荐：AntV G6 v5（强烈推荐）

**理由：**

1. **国内生态**：阿里 AntV 团队维护，文档中文友好，国内社区活跃
2. **功能完备**：力导向布局、自定义节点/边、右键菜单、Minimap、Tooltip 全部内置
3. **性能优秀**：支持大规模图，Canvas 渲染流畅
4. **与 React 集成**：提供 `@antv/g6-react-node` 支持用 React 组件渲染节点
5. **MIT 协议**：免费商用

### 备选：D3.js + 自封装

若 G6 的节点自定义能力不满足需求（如需要复杂的角色卡片交互），可选择 D3.js 作为底层，自行封装组件。但开发成本显著增加。

## 四、设计决策清单

### 4.1 节点层（角色卡片）

| 决策项 | 方案 | 参考来源 |
|--------|------|----------|
| 节点类型 | 自定义卡片节点（React 组件） | G6 React Node |
| 节点尺寸 | 80×70px（缩略版）/ 180×120px（详情版） | 织文 PRD |
| 头像 | 渐变圆形，40px | 织文设计规范 |
| 名字 | 13px，居中，超出截断 | 织文设计规范 |
| 标签 | 底部语义 Tag（男主/女主/反派） | 织文设计规范 |
| 选中态 | 蓝色边框（2px #2563eb） | G6 |
| 拖拽 | 按住节点拖拽调整位置 | G6 |

### 4.2 边层（关系连线）

| 决策项 | 方案 | 参考来源 |
|--------|------|----------|
| 公开关系 | 实线，2px，颜色按关系类型 | G6 |
| 隐藏关系 | 虚线，1.5px，#ff8c00，opacity: 0.6 | G6 |
| 关系标签 | 连线中部显示，背景色块+文字 | G6 Edge Label |
| 边粗细 | 关系强度决定线粗细（强关系 3px，弱关系 1.5px） | D3 最佳实践 |
| 箭头 | 单向关系带箭头（如「爱慕」→） | G6 |

### 4.3 布局层

| 决策项 | 方案 | 参考来源 |
|--------|------|----------|
| 默认布局 | 力导向布局（Force） | G6 |
| 布局参数 | 节点间距 150px，边引力适中 | G6 |
| 手动调整 | 用户拖拽后固定节点位置 | G6 |
| 重新布局 | 点击「重新布局」按钮重新计算 | n8n |
| 隐藏关系开关 | 勾选后隐藏虚线关系 | 织文 PRD |

### 4.4 交互层

| 决策项 | 方案 | 参考来源 |
|--------|------|----------|
| 节点点击 | 选中节点，高亮相关连线 | G6 |
| 节点双击 | 打开角色详情弹窗 | 织文 PRD |
| 右键菜单 | 查看详情 / 编辑关系 / 添加关联 / 删除 | G6 Menu 插件 |
| 画布缩放 | 滚轮缩放 | G6 |
| 画布平移 | 拖拽空白处平移 | G6 |
| 节点拖拽 | 拖拽节点调整位置 | G6 |

## 五、组件拆分建议

```
src/modules/relationship-graph/
├── components/
│   ├── RelationshipCanvas.tsx   # G6 Graph 容器
│   ├── CharacterNode.tsx        # 角色节点（React 组件，供 G6 渲染）
│   ├── RelationEdge.tsx         # 关系边自定义（颜色/虚线/标签）
│   ├── NodeContextMenu.tsx      # 节点右键菜单
│   ├── RelationPanel.tsx        # 右侧关系编辑面板
│   └── MiniMap.tsx              # 小地图（或直接用 G6 Minimap 插件）
├── hooks/
│   ├── useGraph.ts              # G6 Graph 初始化与管理
│   ├── useForceLayout.ts        # 力导向布局控制
│   └── useGraphData.ts          # 图数据转换（角色/关系 → G6 数据）
├── types.ts
├── store.ts
└── ai-prompts.ts
```
