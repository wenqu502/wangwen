## 一、需求背景与范围界定

### 1.1 PRD 原文需求摘录

根据《PRD-网文创作辅助工具-v3.0.md》第 5.2 节「剧情分支树」：

> **核心理念**：用户用自然语言描述故事脉络，AI 生成树形结构。
>
> AI 生成剧情树后，在左侧画布展示：
> - **节点卡片**：显示标题、摘要、状态（已写/待写/进行中）
> - **连线**：实线表示主线，虚线表示分支
> - **颜色**：已写（绿色）、进行中（蓝色）、待写（灰色）、废弃（浅色）
> - **用户可执行的操作**：
>   - 拖拽节点调整位置
>   - 点击节点查看/编辑详情
>   - 拖拽连线调整分支关系
>   - 右键菜单：删除、添加子节点、标记状态
>
> **AI 伏笔追踪**：
> - 自动识别疑似"伏笔"的文本
> - 记录伏笔状态（已埋/已回收/遗忘）
> - 超过设定阈值未回收时主动提醒

#### 3.1.2 架构设计

React Flow 采用分层架构：

```
┌─────────────────────────────────────────┐
│  ReactFlow Provider（全局状态）           │
│  - useNodesState / useEdgesState        │
│  - useReactFlow（ imperative API）       │
├─────────────────────────────────────────┤
│  ReactFlow Container（画布容器）          │
│  - 事件监听（drag/zoom/pan）             │
│  - CSS Transform 变换层                  │
├─────────────────────────────────────────┤
│  Node Layer（节点层）                     │
│  - Node 组件渲染（React 组件）            │
│  - Handle 连接端口                       │
│  - 选中/悬停状态管理                      │
├─────────────────────────────────────────┤
│  Edge Layer（连线层）                     │
│  - SVG Path 渲染（Bezier/SmoothStep/Step）│
│  - Edge Label 标签                       │
│  - 连线交互（hover/click）               │
├─────────────────────────────────────────┤
│  Background / MiniMap / Controls         │
│  - 点状/线状网格背景                      │
│  - 小地图缩略导航                         │
│  - 缩放控制按钮组                         │
└─────────────────────────────────────────┘
```

#### 3.1.3 核心能力逐项评估

**① 节点系统（Node System）**

- **自定义节点类型**：支持完全自定义 React 组件渲染节点。可定义 `storyNode`、`branchNode`、`abandonedNode` 等多种节点类型。
- **节点位置**：通过 `{x, y}` 绝对坐标控制，支持程序式定位和拖拽调整。
- **Handle（连接端口）**：可在节点任意位置定义输入/输出端口，支持多端口。
- **节点状态**：内置 `selected` / `dragging` CSS class，可自定义样式。
- **节点尺寸**：支持动态尺寸，节点宽高可自适应内容。

**② 连线系统（Edge System）**

- **连线类型**：Bezier（默认）、SmoothStep、Step、Straight 四种。
- **自定义连线**：支持自定义 SVG Path 和连线组件。
- **连线标签**：支持在连线中部或任意位置插入标签文字。
- **连线交互**：hover 高亮、click 选中、右键菜单。
- **条件连线**：支持带判断条件的分支连线（如 if/else）。

**③ 交互能力（Interaction）**

- **拖拽**：节点拖拽、连线创建拖拽、画布平移拖拽。
- **缩放**：滚轮缩放、以鼠标位置为中心缩放、程序式缩放。
- **框选**：Shift+拖拽框选多节点，支持多选拖拽。
- **键盘快捷键**：Delete 删除选中、Ctrl+A 全选、Ctrl+Z 撤销（需配合状态管理）。
- **右键菜单**：通过 `onNodeContextMenu` / `onEdgeContextMenu` 实现。

**④ 内置组件（Built-in Components）**

| 组件 | 功能 | 可定制性 |
|------|------|----------|
| `<Background />` | 画布背景（点状/线状/交叉网格） | 间距、颜色、大小可调 |
| `<MiniMap />` | 小地图缩略导航 | 节点样式、视口框样式可定制 |
| `<Controls />` | 缩放控制按钮（+/-/fit） | 按钮样式、位置可定制 |
| `<Panel />` | 固定位置面板容器 | 位置、样式完全自定义 |
| `<NodeToolbar />` | 节点工具栏（悬浮在节点上方） | 内容完全自定义 |
| `<NodeResizer />` | 节点尺寸调整手柄 | 最小/最大尺寸限制 |

**⑤ 状态管理（State Management）**

React Flow 提供两种状态管理方式：

- **受控模式**：通过 `nodes` / `edges` props + `onNodesChange` / `onEdgesChange` 回调管理状态。推荐与 Zustand/Redux 配合使用。
- **非受控模式**：使用 `useNodesState` / `useEdgesState` hook 内部管理状态。

对于织文项目，**推荐受控模式 + Zustand**，因为需要跨模块共享剧情树数据。

#### 3.1.4 与织文需求的匹配度分析

| 织文需求 | React Flow 支持度 | 实现方式 |
|----------|-------------------|----------|
| 节点展示标题+摘要+状态 | 完全支持 | 自定义 Node 组件 |
| 不同状态不同颜色 | 完全支持 | 根据 node.data.status 渲染不同样式 |
| 实线/虚线/点线连线 | 完全支持 | 自定义 Edge 组件，SVG stroke-dasharray |
| 拖拽节点 | 内置 | draggable 属性 |
| 点击节点查看详情 | 完全支持 | onNodeClick 回调打开 Drawer |
| 右键菜单 | 完全支持 | onNodeContextMenu 回调 |
| 画布缩放平移 | 内置 | 滚轮+拖拽 |
| 小地图 | 内置 | `<MiniMap />` 组件 |
| 撤销重做 | 需自行实现 | 配合 Zustand 的 undo/redo middleware |
| 伏笔标记 | 需自行实现 | 节点 data 中存储伏笔数据 |

#### 3.1.5 优缺点分析

| 优点 | 缺点 |
|------|------|
| React 生态原生，与织文技术栈 100% 匹配 | Pro 版高级功能（自动布局、协作编辑）收费 |
| 自定义能力极强，节点/连线可完全自定义 | 学习曲线中等，API 较复杂 |
| 内置功能完备（拖拽/缩放/小地图/控制按钮） | 100+ 节点时可能需要性能优化（虚拟化） |
| MIT 协议，免费商用 | 撤销重做需自行集成 |
| 社区活跃，文档完善，示例丰富 | |
| 支持 TypeScript，类型定义完整 | |

|------|-------------|------|

### 3.2 Rete.js —— 节点编辑器框架

#### 3.2.1 项目基本信息

| 项 | 内容 |
|----|------|
| 官网 | https://retejs.org |
| GitHub | https://github.com/retejs/rete |
| Stars | 6.8k |
| 最新版本 | v2.x |
| 技术栈 | TypeScript（框架无关，支持 React/Vue/Angular） |
| 开源协议 | MIT |
| 维护状态 | 活跃，社区驱动 |

#### 3.2.2 核心特点

- **框架无关**：核心引擎不依赖任何 UI 框架，通过插件机制支持 React/Vue/Angular。
- **数据流驱动**：节点之间的数据流向是核心设计，适合数据管道场景。
- **插件丰富**：Area 插件（画布）、Connection 插件（连线）、ContextMenu 插件（右键菜单）等。

#### 3.2.3 与织文需求的匹配度

React Flow 相比 Rete.js 的优势：
- React Flow 是 React 原生，Rete.js 需要额外的 React 渲染插件层。
- React Flow 的节点自定义更直观（直接写 React 组件），Rete.js 需要理解其节点/控制/连接抽象。
- Rete.js 更适合数据流处理场景（如音频处理、图像处理），织文是内容展示+编辑，React Flow 更匹配。

#### 3.2.4 评分

| 维度 | 得分 | 说明 |
|------|------|------|
| 技术栈匹配度 | 3 | 需额外插件层 |
| 自定义能力 | 4 | 高但学习成本高 |
| 内置功能完备度 | 3.5 | 插件分散 |
| 性能表现 | 4 | 良好 |
| 社区活跃度 | 3 | 中等 |
| 许可证 | 5 | MIT |

## 五、推荐方案：React Flow 详细设计

### 5.1 技术架构

```
src/modules/plot-tree/
├── components/
│   ├── PlotCanvas.tsx              # ReactFlow 容器画布
│   ├── nodes/
│   │   ├── StoryNode.tsx           # 剧情节点（已完成/进行中/待处理）
│   │   ├── BranchNode.tsx          # 分支节点
│   │   ├── AbandonedNode.tsx       # 废弃节点
│   │   └── NodeContextMenu.tsx     # 节点右键菜单
│   ├── edges/
│   │   ├── MainEdge.tsx            # 主线连线（实线）
│   │   ├── BranchEdge.tsx          # 分支连线（虚线）
│   │   └── AbandonedEdge.tsx       # 废弃连线（点线）
│   ├── overlays/
│   │   ├── CanvasControls.tsx      # 缩放控制（或直接用 ReactFlow Controls）
│   │   ├── MiniMapPanel.tsx        # 小地图
│   │   └── NodeToolbar.tsx         # 节点悬浮工具栏
│   └── panels/
│       ├── NodeDetailDrawer.tsx    # 节点详情编辑面板
│       └── ForesightPanel.tsx      # 伏笔追踪面板
├── hooks/
│   ├── usePlotTree.ts              # 剧情树核心状态管理
│   ├── useNodeDrag.ts              # 节点拖拽逻辑
│   ├── useEdgeConnect.ts           # 连线创建逻辑
│   ├── useNodeContextMenu.ts       # 右键菜单逻辑
│   └── useUndoRedo.ts              # 撤销重做逻辑
├── utils/
│   ├── layout.ts                   # 自动布局算法（初始布局）
│   └── exportImport.ts             # JSON 导入导出
```

### 5.3 Zustand Store 设计

```typescript
// store.ts

import { create } from 'zustand'
import { persist } from 'zustand/middleware'
import { immer } from 'zustand/middleware/immer'
import type { PlotNode, PlotEdge, PlotTreeState } from './types'

interface PlotTreeStore extends PlotTreeState {
  // 节点操作
  addNode: (node: PlotNode) => void
  updateNode: (id: string, updates: Partial Partial<PlotNode['data']>) => void
  removeNode: (id: string) => void
  moveNode: (id: string, position: { x: number; y: number }) => void
  setNodeStatus: (id: string, status: PlotNode['data']['status']) => void

  // 连线操作
  addEdge: (edge: PlotEdge) => void
  removeEdge: (id: string) => void

  // 选择态
  selectNode: (id: string | null) => void

  // 视图操作
  setViewport: (viewport: PlotTreeState['viewport']) => void

  // 导入导出
  exportToJSON: () => string
  importFromJSON: (json: string) => void
}

export const usePlotTreeStore = create create<PlotTreeStore>()(
  persist(
    immer((set, get) => ({
      nodes: [],
      edges: [],
      selectedNodeId: null,
      viewport: { x: 0, y: 0, zoom: 1 },

      addNode: (node) =>
        set((state) => {
          state.nodes.push(node)
        }),

      updateNode: (id, updates) =>
        set((state) => {
          const node = state.nodes.find((n) => n.id === id)
          if (node) {
            Object.assign(node.data, updates)
          }
        }),

      removeNode: (id) =>
        set((state) => {
          state.nodes = state.nodes.filter((n) => n.id !== id)
          state.edges = state.edges.filter(
            (e) => e.source !== id && e.target !== id
          )
        }),

      moveNode: (id, position) =>
        set((state) => {
          const node = state.nodes.find((n) => n.id === id)
          if (node) {
            node.position = position
          }
        }),

      setNodeStatus: (id, status) =>
        set((state) => {
          const node = state.nodes.find((n) => n.id === id)
          if (node) {
            node.data.status = status
            // 废弃节点自动改为 abandonedNode 类型
            if (status === 'abandoned') {
              node.type = 'abandonedNode'
            }
          }
        }),

      addEdge: (edge) =>
        set((state) => {
          state.edges.push(edge)
        }),

      removeEdge: (id) =>
        set((state) => {
          state.edges = state.edges.filter((e) => e.id !== id)
        }),

      selectNode: (id) =>
        set((state) => {
          state.selectedNodeId = id
        }),

      setViewport: (viewport) =>
        set((state) => {
          state.viewport = viewport
        }),

      exportToJSON: () => JSON.stringify({
        nodes: get().nodes,
        edges: get().edges
      }),

      importFromJSON: (json) => {
        const data = JSON.parse(json)
        set((state) => {
          state.nodes = data.nodes
          state.edges = data.edges
        })
      }
    })),
    {
      name: 'plot-tree-storage',
      partialize: (state) => ({
        nodes: state.nodes,
        edges: state.edges,
        viewport: state.viewport
      })
    }
  )
)
```

### 5.7 撤销重做实现（基于 Zustand + Immer）

```typescript
// hooks/useUndoRedo.ts

import { useCallback, useRef } from 'react'
import type { PlotTreeState } from '../types'

interface HistoryState {
  past: PlotTreeState[]
  present: PlotTreeState
  future: PlotTreeState[]
}

const MAX_HISTORY = 50

export function useUndoRedo(
  getState: () => PlotTreeState,
  setState: (state: PlotTreeState) => void
) {
  const historyRef = useRefRef<HistoryState>({
    past: [],
    present: getState(),
    future: []
  })

  const saveHistory = useCallback(() => {
    const current = getState()
    const history = historyRef.current

    history.past.push(history.present)
    if (history.past.length > MAX_HISTORY) {
      history.past.shift()
    }
    history.present = current
    history.future = []
  }, [getState])

  const undo = useCallback(() => {
    const history = historyRef.current
    if (history.past.length === 0) return

    history.future.unshift(history.present)
    history.present = history.past.pop()!
    setState(history.present)
  }, [setState])

  const redo = useCallback(() => {
    const history = historyRef.current
    if (history.future.length === 0) return

    history.past.push(history.present)
    history.present = history.future.shift()!
    setState(history.present)
  }, [setState])

  return { saveHistory, undo, redo }
}
```

### 5.8 性能优化策略

| 优化点 | 方案 | 触发条件 |
|--------|------|----------|
| 节点虚拟化 | 仅渲染视口内节点 + 缓冲区 | 节点数 > 100 |

| Canvas 渲染 | 超大图时切换为 Canvas 模式 | 节点数 > 500 |

| 风险 | 影响 | 概率 | 应对方案 |
|------|------|------|----------|
| 100+ 节点性能下降 | 拖拽卡顿 | 中 | 实现虚拟化，仅渲染视口内节点 |
