# 剧情分支树可视化编辑器 —— 开源项目调研文档

> 文档类型：前端开发前期调研

> 调研模块：剧情分支树（Plot Branch Tree）可视化编辑器
> 调研日期：2026-05-04
> 调研人：program-frontend-design-expert

---

## 一、调研背景

织文 WangWen 的核心功能之一是「剧情分支树」，用户通过自然语言与 AI 对话，AI 生成剧情骨架后以可视化方式展示。用户需要在画布上：

- 查看主线/分支节点及其状态（已完成/进行中/待处理/废弃）
- 拖拽节点调整位置
- 拖拽连线调整分支关系
- 右键节点执行操作（编辑/删除/添加子节点）
- 缩放和平移画布以浏览大型剧情树

这本质上是一个**节点编辑器（Node Editor）** / **工作流构建器（Workflow Builder）**，与当前主流的自动化工作流工具、AI 管道工具在交互模式上高度相似。

---

## 二、调研目标

1. 找到业界成熟的节点编辑器/工作流构建器的开源实现
2. 分析其 UI 设计模式，提取可直接借鉴的布局、组件、交互方案
3. 评估技术可行性，为织文的前端技术选型提供依据
4. 输出可直接指导开发的设计决策清单

## 三、候选项目分析

### 3.1 n8n —— 工作流自动化平台

**项目信息**

| 项 | 内容 |
|----|------|
| 官网 | https://n8n.io |
| GitHub | https://github.com/n8n-io/n8n |
| Stars | 55.8k+ |
| 技术栈 | Vue 3 + TypeScript |
| 开源协议 | fair-code（社区版免费） |
| 定位 | 节点式工作流自动化工具（Zapier 开源替代） |

**核心设计模式**

1. **画布（Canvas）**
   - 浅灰色点状网格背景，间距约 20px
   - 节点通过绝对定位放置在画布上
   - 支持拖拽平移（空格+拖拽 / 中键拖拽）
   - 支持缩放（滚轮 / 右下角控制按钮）

2. **节点（Node）**
   - 顶部彩色标题栏（根据节点类型着色）
   - 标题栏左侧为节点图标，右侧为状态指示
   - 下方为参数预览区（收起态只显示标题）
   - 左侧输入端口（Input），右侧输出端口（Output）
   - 端口为小圆点，hover 时放大

3. **连线（Edge）**
   - 贝塞尔曲线（cubic-bezier）连接输出端口到输入端口
   - 连线颜色与源节点主题色一致
   - hover 连线时高亮，显示删除按钮

4. **交互**
   - 拖拽节点：任意位置拖拽调整
   - 拖拽连线：从端口拖出创建新连线
   - 框选：支持多节点框选批量操作
   - 右键菜单：编辑/复制/删除/禁用

5. **辅助元素**
   - 右下角缩放控制（+ / 百分比 / - / 全屏）
   - 左下角小地图（minimap），显示整体布局和当前视口
   - 执行时节点高亮动画（绿色脉冲）

**对织文的借鉴点**

| 借鉴项 | 具体应用 |
|--------|----------|
| 点状网格画布 | 剧情树画布背景，提供空间参考但不干扰内容 |
| 节点顶部标题栏 | 剧情节点头部显示标题+状态标签，颜色区分状态 |
| 输入/输出端口 | 节点左侧（入）右侧（出），用于拖拽创建分支连线 |
| 贝塞尔曲线连线 | 主线/分支/废弃关系的连线表达，比直线更自然 |
| 缩放控制 + 小地图 | 大型剧情树（50+ 节点）的导航必备 |
| 右键菜单 | 节点操作（编辑详情/标记状态/删除） |

### 3.2 ComfyUI —— AI 图像生成工作流工具

**项目信息**

| 项 | 内容 |
|----|------|
| 官网 | https://comfy.org |
| GitHub | https://github.com/comfyanonymous/ComfyUI |
| Stars | 70k+ |
| 技术栈 | Python 后端 + 原生 JS 前端（Canvas + WebGL） |
| 开源协议 | GPL-3.0 |
| 定位 | 节点式 Stable Diffusion 工作流编辑器 |

**核心设计模式**

1. **节点设计**
   - 圆角矩形卡片，白色背景
   - 顶部标题栏：深色渐变背景（节点类型色）+ 白色文字
   - 标题栏左侧有小圆点可折叠/展开节点
   - 内容区：参数输入表单（文本/数字/下拉/文件）
   - 输入端口：左侧纵向排列，带类型标识
   - 输出端口：右侧纵向排列

2. **连线设计**
   - 贝塞尔曲线，粗线条（约 3-4px）
   - 连线颜色根据数据类型变化（如 IMAGE 为紫色，LATENT 为粉色）
   - 正在拖拽的连线显示虚线预览
   - 不兼容的端口 hover 时连线变红

3. **画布交互**
   - 拖拽空白处平移画布
   - 滚轮缩放（以鼠标位置为中心）
   - 节点拖拽时有轻微阴影提升
   - 选中节点显示蓝色边框

4. **分组（Group）**
   - 可将多个节点框选并创建分组
   - 分组有标题和背景色块
   - 分组可整体移动

5. **工作流管理**
   - 支持多工作流标签页切换
   - 工作流可导出为 JSON / PNG（含元数据）
   - 支持模板库

**对织文的借鉴点**

| 借鉴项 | 具体应用 |
|--------|----------|
| 节点标题栏彩色渐变 | 剧情节点头部用状态色（绿/蓝/灰/虚线灰）区分 |
| 端口纵向排列 | 若一个节点有多输入/多输出（如分支汇聚点）可用 |
| 拖拽连线虚线预览 | 用户拖拽创建分支时的视觉反馈 |
| 分组功能 | 剧情「阶段」分组（如「第一卷」「第二卷」） |
| 工作流标签页 | 对应织文的「多作品切换」或「多剧情线切换」 |

### 3.3 React Flow —— React 节点编辑器库

**项目信息**

| 项 | 内容 |
|----|------|
| 官网 | https://reactflow.dev |
| GitHub | https://github.com/xyflow/xyflow |
| Stars | 28k+ |
| 技术栈 | React + TypeScript |
| 开源协议 | MIT（Pro 版额外功能收费） |
| 定位 | 用于构建节点编辑器的 React 组件库 |

**核心设计模式**

1. **架构设计**
   - `ReactFlow` 容器组件包裹整个画布
   - `Node` 组件可完全自定义（用 React 组件渲染）
   - `Edge` 支持多种类型：Bezier / SmoothStep / Step / Straight
   - 内置 `MiniMap`、`Controls`、`Background` 组件

2. **节点系统**
   - 节点位置通过 `{x, y}` 绝对坐标控制
   - 支持自定义节点类型（如 `default` / `input` / `output` / 自定义）
   - 节点可定义多个 `Handle`（连接端口），位置自由
   - 选中/悬停状态通过 CSS class 控制

3. **连线系统**
   - 支持 4 种连线类型：Bezier（默认）、SmoothStep、Step、Straight
   - 连线可自定义颜色、粗细、标签
   - 支持条件连线（带判断条件的分支）
   - 连线中间可插入标签文字

4. **交互能力**
   - 内置拖拽、缩放、平移
   - 支持框选（selection）
   - 支持键盘快捷键（Delete 删除选中、Ctrl+A 全选等）
   - 支持撤销/重做（通过 onNodesChange/onEdgesChange 配合状态管理）

5. **内置组件**
   - `<Background />`：可配置点状/线状网格
   - `<MiniMap />`：小地图，可配置节点缩略样式
   - `<Controls />`：缩放控制按钮组
   - `<Panel />`：固定位置面板（如属性编辑面板）

**对织文的借鉴点**

| 借鉴项 | 具体应用 |
|--------|----------|
| 自定义节点类型 | 定义 `storyNode`、`branchNode`、`abandonedNode` 等类型 |
| Bezier 连线 | 剧情主线/分支的连线表达 |
| MiniMap 组件 | 大型剧情树的整体导航 |
| Controls 组件 | 缩放控制，直接复用 |
| Background 网格 | 画布背景，可配置间距和颜色 |
| 撤销/重做 | 剧情树编辑操作的历史管理 |

## 四、技术选型建议

### 4.1 直接采用 React Flow（强烈推荐）

**理由：**

1. **技术栈匹配**：织文使用 React 18 + TypeScript + Vite，React Flow 是 React 生态原生库
2. **自定义能力极强**：节点、连线、手柄均可完全自定义 React 组件，与设计规范 100% 对接
3. **内置功能完备**：拖拽、缩放、小地图、控制按钮、框选、撤销重做均内置
4. **MIT 协议**：免费商用，无授权风险
5. **社区活跃**：大量示例和教程，xyflow 团队持续维护

**Pro 版评估：**

React Flow Pro 提供额外功能（如自动布局 `Layout`、协作编辑等），初期可先使用免费版，需要时再评估 Pro 版。

### 4.2 不直接采用 n8n / ComfyUI 的原因

| 项目 | 不采用原因 |
|------|-----------|
| n8n | 是完整应用而非组件库，耦合 Vue 3 和后端逻辑，无法直接嵌入 React 项目 |
| ComfyUI | 是 Python 后端驱动的完整应用，前端为原生 JS，与 React 技术栈不匹配 |

**但它们的设计模式（节点样式、连线方式、画布交互）可以直接借鉴并迁移到 React Flow 上实现。**

## 五、设计决策清单

### 5.1 画布层

| 决策项 | 方案 | 参考来源 |
|--------|------|----------|
| 背景 | 浅灰点状网格，间距 20px | n8n |
| 画布尺寸 | 最小 800×600，支持无限扩展 | React Flow |
| 缩放范围 | 0.25x ~ 2x，默认 1x | React Flow + n8n |
| 平移方式 | 空格+拖拽 / 中键拖拽 / 滚轮平移（可选） | n8n |
| 小地图 | 左下角，140×90px，显示节点缩略和视口框 | React Flow MiniMap |
| 缩放控制 | 右下角，+ / 百分比 / - / 全屏按钮 | React Flow Controls |

### 5.2 节点层

| 决策项 | 方案 | 参考来源 |
|--------|------|----------|
| 节点尺寸 | 宽度 180px，高度自适应 | ComfyUI |
| 节点圆角 | 8px | ComfyUI |
| 节点头部 | 顶部彩色标题栏（状态色）+ 白色文字 + 状态标签 | ComfyUI + n8n |
| 节点内容 | 摘要文字，12px，最多 2 行 | 织文 PRD |
| 输入端口 | 左侧居中，10px 圆点，白色底+彩色边 | React Flow Handle |
| 输出端口 | 右侧居中，10px 圆点，白色底+彩色边 | React Flow Handle |
| 选中态 | 蓝色边框（2px #2563eb）+ 阴影提升 | React Flow |
| 废弃态 | 整体 opacity: 0.6，标题栏灰色，标题删除线 | n8n（禁用节点样式） |
| 拖拽反馈 | 节点跟随鼠标，释放后定位 | React Flow |

### 5.3 连线层

| 决策项 | 方案 | 参考来源 |
|--------|------|----------|
| 连线类型 | Bezier（贝塞尔曲线） | React Flow + n8n |
| 主线样式 | 实线，2.5px，#86909c | n8n |
| 分支样式 | 虚线，2px，#2563eb，stroke-dasharray: 6 4 | n8n（分支连线） |
| 废弃连线 | 点线，1.5px，#ff8c00，opacity: 0.6 | ComfyUI |
| 连线标签 | 连线中部显示关系标签（如「分支A」「隐藏关联」） | React Flow Edge Label |
| 拖拽创建 | 从输出端口拖出虚线，连接到输入端口创建新关系 | ComfyUI + React Flow |

### 5.4 交互层

| 决策项 | 方案 | 参考来源 |
|--------|------|----------|
| 节点点击 | 选中节点，右侧属性面板显示详情 | n8n |
| 节点双击 | 打开节点详情弹窗 | n8n |
| 右键菜单 | 编辑 / 添加子节点 / 标记状态 / 删除 | n8n |
| 框选 | Shift+拖拽框选多节点，批量移动/删除 | React Flow |
| 删除 | Delete 键删除选中节点/连线 | React Flow |
| 撤销/重做 | Ctrl+Z / Ctrl+Y，通过 React Flow 状态变更实现 | React Flow |

## 七、风险与注意事项

1. **性能风险**：React Flow 在 100+ 节点时可能出现性能下降，需测试并考虑虚拟化
2. **移动端适配**：React Flow 主要面向桌面端，移动端操作体验需额外优化
3. **Pro 版依赖**：若后续需要自动布局（Dagre/ELK），可能需要引入 Pro 版或自行实现
4. **Zustand 状态同步**：React Flow 内部状态与 Zustand 的同步需要仔细设计，避免不一致
