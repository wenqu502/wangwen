## 一、调研背景

织文 WangWen 的「体系管理工具」是一个完全抽象化的设定设计工具，不限定任何题材（修炼/魔法/货币/爵位等均可）。用户通过 AI 对话创建分层分级的结构化设定，系统以树形或层级可视化展示。核心需求：

- AI 对话式创建体系（如「魔法体系分元素系和召唤系，每系5个等级」）
- 可视化展示体系结构（树形 / 层级 / 表格）
- 规则配置与冲突检测
- 支持多体系并行管理

## 二、候选项目分析

### 2.1 React Arborist —— React 树形组件

| 项 | 内容 |
|----|------|
| GitHub | https://github.com/brimdata/react-arborist |
| Stars | 3k+ |
| 技术栈 | React |
| 开源协议 | MIT |
| 定位 | 可拖拽排序的树形组件 |

**核心能力**
- 拖拽节点调整层级和顺序
- 展开/折叠节点
- 虚拟滚动（支持大量节点）
- 多选、框选
- 搜索过滤

**适用性**：适合「体系树」的层级编辑，如魔法体系的系→等级结构

## 三、技术选型建议

### 推荐：React Arborist（强烈推荐）

**理由：**

1. **React 原生**：与织文技术栈完全匹配
2. **拖拽体验优秀**：拖拽节点调整层级非常流畅
3. **虚拟滚动**：支持大量节点不卡顿
4. **高度可定制**：节点完全自定义 React 组件
5. **活跃维护**：社区活跃，持续更新

### 备选：Ant Design Tree

若项目已使用 Ant Design，可直接使用其 Tree 组件，减少额外依赖。但拖拽体验不如 Arborist。

## 四、设计决策清单

### 4.1 体系树展示

| 决策项 | 方案 |
|--------|------|
| 组件 | React Arborist |
| 节点样式 | 自定义节点：图标 + 名称 + 等级标识 |
| 层级缩进 | 24px/级 |
| 展开/折叠 | 左侧箭头，支持全部展开/折叠 |
| 拖拽 | 按住节点拖拽调整层级和顺序 |
| 多选 | 支持多选节点批量操作 |

### 4.2 体系详情面板

| 决策项 | 方案 |
|--------|------|
| 打开方式 | 点击节点右侧展开详情面板 |
| 面板宽度 | 320px |
| 内容 | 等级名称 / 描述 / 晋升条件 / 能力列表 / 限制规则 |
| 编辑 | 点击字段进入行内编辑 |

### 4.3 规则配置

| 决策项 | 方案 |
|--------|------|
| 规则列表 | 表格展示所有规则 |
| 规则字段 | 规则描述 / 严重程度（硬性/软性）/ 例外情况 |
| 冲突标记 | 检测到冲突的规则标红，显示冲突详情 |
