## 一、需求背景与范围界定

### 1.1 PRD 原文需求摘录

根据《PRD-网文创作辅助工具-v3.0.md》第 5.4 节「体系管理工具」：

> **核心理念**：完全抽象的体系设计工具，不限定任何题材。
>
> **「体系」定义**：作品中任何**分层分级、有规则约束的结构化设定**。包括但不限于：
> - 力量体系：修炼境界、魔法等级、异能分类
> - 社会体系：贵族爵位、官僚等级、帮派辈分
> - 经济体系：货币兑换、资源稀有度、物价体系
> - 道具体系：装备品质、武器分级、丹药品阶
> - 知识体系：学科分类、技能树、科技树
> - 生物体系：物种等级、进化链、食物链
>
> **AI 对话式创建体系**：
> - 用户自然语言描述体系结构
> - AI 生成分支 + 等级 + 默认字段
> - 支持后续追加分支（如新增「时间系」）
> - AI 自动给出平衡性建议（如「时间系容易破坏剧情平衡，建议加入限制条件」）
>
> **体系抽象数据结构**：
> - 体系名称、描述
> - 分支列表（branch）：每个分支含多个等级（level）
> - 等级字段：rank、name、description、promotionCondition、abilities、restrictions
> - 规则列表（rule）：desc、severity（硬性/软性）、exceptions
>
> **AI 规则冲突检测**：
> - 用户撰写内容时自动检测是否违反体系规则
> - 冲突时给出处理建议（修改文本/添加例外/修改规则/忽略）

根据《PRD-网文创作辅助工具-v2.0.md》补充界面需求：

> - 世界观设定库：设定分类 CRUD + 修炼体系编辑器 + 年表
> - 五大分类可正常录入，修炼体系编辑器可渲染阶梯图

| 项 | 内容 |
|----|------|
| GitHub | https://github.com/brimdata/react-arborist |
| Stars | 3.5k+ |
| 最新版本 | v3.x |
| 技术栈 | React + TypeScript |
| 开源协议 | MIT |

| 维护团队 | Brim Data |

**核心能力详解**

1. **拖拽排序**：
   - 拖拽节点调整顺序（同级排序）

   - `renderRow`：自定义整行渲染
   - 完全控制节点外观

## 四、综合评分与选型决策

### 4.1 评分矩阵

| 维度（权重） | React Arborist | Ant Design Tree | @tanstack/react-tree | react-complex-tree | rc-tree |
|-------------|---------------|-----------------|---------------------|-------------------|---------|
| 技术栈匹配度（25%） | 5 | 5 | 5 | 5 | 5 |
| 树形交互能力（25%） | 5 | 4 | 3 | 5 | 4 |
| 自定义能力（20%） | 5 | 4 | 5 | 5 | 4 |
| 性能表现（15%） | 5 | 3 | 5 | 3 | 3 |
| 社区活跃度（10%） | 3 | 5 | 3 | 3 | 4 |
| 许可证（5%） | 5 | 5 | 5 | 5 | 5 |
| **加权总分** | **4.65** | **4.20** | **4.15** | **4.25** | **4.05** |

**评分细则**

- **React Arborist（4.65/5）**：
  - 拖拽排序和改变层级体验最佳
  - 虚拟滚动内置，性能优秀
  - 完全自定义节点渲染
  - 轻量且现代
  - 社区相对小但活跃

- **Ant Design Tree（4.20/5）**：
  - 社区最大，文档最完善
  - 但拖拽体验一般，虚拟滚动需额外配置
  - 包体积大

- **@tanstack/react-tree（4.15/5）**：
  - 虚拟滚动性能最佳
  - 但较新，拖拽等交互需自行实现
  - 适合未来演进

- **react-complex-tree（4.25/5）**：
  - 功能全面，但无虚拟滚动
  - 50+ 节点可能卡顿

- **rc-tree（4.05/5）**：
  - 底层灵活，但开发成本高

### 4.2 选型结论

**主选方案：React Arborist**

理由：
1. **拖拽体验最佳**：调整分支/等级顺序和层级的核心交互最流畅
2. **虚拟滚动内置**：支持大型体系不卡顿
3. **完全自定义**：节点渲染完全控制，匹配织文设计规范
4. **轻量现代**：~20KB gzip，不引入冗余依赖
5. **TypeScript 原生**：类型定义完善

**备选方案：Ant Design Tree**

若项目已引入 Ant Design，或团队对 Ant Design 更熟悉时选用。

**未来预留：@tanstack/react-tree**

若 TanStack 生态在织文项目中广泛使用时考虑统一技术栈。

## 六、Zustand Store 设计

```typescript
// src/modules/system-builder/store.ts

import { create } from 'zustand'
import { immer } from 'zustand/middleware/immer'
import { persist } from 'zustand/middleware'
import {
  System,
  SystemBranch,
  SystemLevel,
  SystemRule,
  RuleConflict,
  TimelineEvent,
  SystemTreeNode,
} from './types'

interface SystemBuilderStore extends SystemBuilderState {
  // 体系 CRUD
  addSystem: (system: System) => void
  updateSystem: (id: string, updates: Partial<System>) => void
  removeSystem: (id: string) => void
  setActiveSystem: (id: string | null) => void

  // 分支 CRUD
  addBranch: (systemId: string, branch: SystemBranch) => void
  updateBranch: (systemId: string, branchId: string, updates: Partial<SystemBranch>) => void
  removeBranch: (systemId: string, branchId: string) => void

  // 等级 CRUD
  addLevel: (systemId: string, branchId: string, level: SystemLevel) => void
  updateLevel: (
    systemId: string,
    branchId: string,
    levelId: string,
    updates: Partial<SystemLevel>
  ) => void
  removeLevel: (systemId: string, branchId: string, levelId: string) => void

  // 规则 CRUD
  addRule: (systemId: string, rule: SystemRule) => void
  updateRule: (systemId: string, ruleId: string, updates: Partial<SystemRule>) => void
  removeRule: (systemId: string, ruleId: string) => void

  // 树形状态
  selectNode: (id: string | null) => void
  toggleExpanded: (id: string) => void
  expandAll: () => void
  collapseAll: () => void

  // 冲突
  setConflicts: (conflicts: RuleConflict[]) => void
  resolveConflict: (conflictId: string, resolution: string) => void

  // 年表
  addTimelineEvent: (event: TimelineEvent) => void
  updateTimelineEvent: (id: string, updates: Partial<TimelineEvent>) => void
  removeTimelineEvent: (id: string) => void

  // 计算属性
  getActiveSystem: () => System | undefined
  getTreeData: () => SystemTreeNode[]

  // 导入导出
  exportToJSON: () => string
  importFromJSON: (json: string) => void
}

export const useSystemBuilderStore = create<SystemBuilderStore>()(
  immer(
    persist(
      (set, get) => ({
        // 初始状态
        systems: [],
        activeSystemId: null,
        selectedNodeId: null,
        expandedNodeIds: [],
        conflicts: [],
        timeline: [],

        // 体系 CRUD
        addSystem: (system) =>
          set((state) => {
            state.systems.push(system)
            if (!state.activeSystemId) {
              state.activeSystemId = system.id
            }
          }),

        updateSystem: (id, updates) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === id)
            if (sys) {
              Object.assign(sys, updates, { updatedAt: new Date().toISOString() })
            }
          }),

        removeSystem: (id) =>
          set((state) => {
            state.systems = state.systems.filter((s) => s.id !== id)
            if (state.activeSystemId === id) {
              state.activeSystemId = state.systems[0]?.id || null
            }
          }),

        setActiveSystem: (id) =>
          set((state) => {
            state.activeSystemId = id
            state.selectedNodeId = null
          }),

        // 分支 CRUD
        addBranch: (systemId, branch) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === systemId)
            if (sys) {
              sys.branches.push(branch)
            }
          }),

        updateBranch: (systemId, branchId, updates) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === systemId)
            const branch = sys?.branches.find((b) => b.id === branchId)
            if (branch) {
              Object.assign(branch, updates)
            }
          }),

        removeBranch: (systemId, branchId) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === systemId)
            if (sys) {
              sys.branches = sys.branches.filter((b) => b.id !== branchId)
            }
          }),

        // 等级 CRUD
        addLevel: (systemId, branchId, level) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === systemId)
            const branch = sys?.branches.find((b) => b.id === branchId)
            if (branch) {
              branch.levels.push(level)
              branch.levels.sort((a, b) => a.rank - b.rank)
            }
          }),

        updateLevel: (systemId, branchId, levelId, updates) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === systemId)
            const branch = sys?.branches.find((b) => b.id === branchId)
            const level = branch?.levels.find((l) => l.id === levelId)
            if (level) {
              Object.assign(level, updates)
            }
          }),

        removeLevel: (systemId, branchId, levelId) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === systemId)
            const branch = sys?.branches.find((b) => b.id === branchId)
            if (branch) {
              branch.levels = branch.levels.filter((l) => l.id !== levelId)
            }
          }),

        // 规则 CRUD
        addRule: (systemId, rule) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === systemId)
            if (sys) {
              sys.rules.push(rule)
            }
          }),

        updateRule: (systemId, ruleId, updates) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === systemId)
            const rule = sys?.rules.find((r) => r.id === ruleId)
            if (rule) {
              Object.assign(rule, updates)
            }
          }),

        removeRule: (systemId, ruleId) =>
          set((state) => {
            const sys = state.systems.find((s) => s.id === systemId)
            if (sys) {
              sys.rules = sys.rules.filter((r) => r.id !== ruleId)
            }
          }),

        // 树形状态
        selectNode: (id) =>
          set((state) => {
            state.selectedNodeId = id
          }),

        toggleExpanded: (id) =>
          set((state) => {
            const index = state.expandedNodeIds.indexOf(id)
            if (index === -1) {
              state.expandedNodeIds.push(id)
            } else {
              state.expandedNodeIds.splice(index, 1)
            }
          }),

        expandAll: () =>
          set((state) => {
            const ids: string[] = []
            state.systems.forEach((sys) => {
              ids.push(sys.id)
              sys.branches.forEach((branch) => {
                ids.push(branch.id)
                branch.levels.forEach((level) => ids.push(level.id))
              })
            })
            state.expandedNodeIds = ids
          }),

        collapseAll: () =>
          set((state) => {
            state.expandedNodeIds = []
          }),

        // 冲突
        setConflicts: (conflicts) =>
          set((state) => {
            state.conflicts = conflicts
          }),

        resolveConflict: (conflictId, resolution) =>
          set((state) => {
            const conflict = state.conflicts.find((c) => c.id === conflictId)
            if (conflict) {
              conflict.resolved = true
              conflict.resolution = resolution
            }
          }),

        // 年表
        addTimelineEvent: (event) =>
          set((state) => {
            state.timeline.push(event)
            state.timeline.sort((a, b) => a.year - b.year)
          }),

        updateTimelineEvent: (id, updates) =>
          set((state) => {
            const event = state.timeline.find((e) => e.id === id)
            if (event) {
              Object.assign(event, updates)
            }
          }),

        removeTimelineEvent: (id) =>
          set((state) => {
            state.timeline = state.timeline.filter((e) => e.id !== id)
          }),

        // 计算属性
        getActiveSystem: () => {
          return get().systems.find((s) => s.id === get().activeSystemId)
        },

        getTreeData: () => {
          return get().systems.map((system) => ({
            id: system.id,
            name: system.name,
            type: 'system' as const,
            data: system,
            children: system.branches.map((branch) => ({
              id: branch.id,
              name: branch.name,
              type: 'branch' as const,
              data: branch,
              children: branch.levels.map((level) => ({
                id: level.id,
                name: `T${level.rank} ${level.name}`,
                type: 'level' as const,
                data: level,
              })),
            })),
          }))
        },

        // 导入导出
        exportToJSON: () => {
          const { systems, timeline } = get()
          return JSON.stringify({ systems, timeline }, null, 2)
        },

        importFromJSON: (json) =>
          set((state) => {
            try {
              const data = JSON.parse(json)
              state.systems = data.systems || []
              state.timeline = data.timeline || []
            } catch (e) {
              console.error('Import failed:', e)
            }
          }),
      }),
      {
        name: 'wangwen-system-builder',
        partialize: (state) => ({
          systems: state.systems,
          activeSystemId: state.activeSystemId,
          timeline: state.timeline,
          conflicts: state.conflicts,
        }),
      }
    )
  )
)
```

```tsx
import React, { useState } from 'react'
import { useSystemBuilderStore } from '../store'

export const NodeDetailPanel: React.FC = () => {
  const {
    selectedNodeId,
    getActiveSystem,
    updateLevel,
    updateBranch,
    updateSystem,
  } = useSystemBuilderStore()

  const system = getActiveSystem()
  const [activeTab, setActiveTab] = useState('basic')

  if (!system || !selectedNodeId) {
    return (
      <div className="detail-panel-empty">
        <p>选择一个节点查看详情</p>
      </div>
    )
  }

  // 查找选中的节点
  let node: any = null
  let nodeType = ''

  if (system.id === selectedNodeId) {
    node = system
    nodeType = 'system'
  } else {
    for (const branch of system.branches) {
      if (branch.id === selectedNodeId) {
        node = branch
        nodeType = 'branch'
        break
      }
      for (const level of branch.levels) {
        if (level.id === selectedNodeId) {
          node = level
          nodeType = 'level'
          break
        }
      }
    }
  }

  if (!node) return null

  return (
    <div className="node-detail-panel">
      <div className="panel-header">
        <h3>{node.name}</h3>
        <span className={`type-badge ${nodeType}`}>{nodeType}</span>
      </div>

      <div className="panel-tabs">
        <button
          className={activeTab === 'basic' ? 'active' : ''}
          onClick={() => setActiveTab('basic')}
        >
          基础信息
        </button>
        {nodeType === 'level' && (
          <button
            className={activeTab === 'abilities' ? 'active' : ''}
            onClick={() => setActiveTab('abilities')}
          >
            能力
          </button>
        )}
        {nodeType === 'system' && (
          <button
            className={activeTab === 'rules' ? 'active' : ''}
            onClick={() => setActiveTab('rules')}
          >
            规则
          </button>
        )}
      </div>

      <div className="panel-body">
        {activeTab === 'basic' && (
          <div className="form-fields">
            <div className="field">
              <label>名称</label>
              <input
                value={node.name}
                onChange={(e) => {
                  if (nodeType === 'system') updateSystem(node.id, { name: e.target.value })
                  if (nodeType === 'branch') updateBranch(system.id, node.id, { name: e.target.value })
                  if (nodeType === 'level')
                    updateLevel(system.id, node.branchId, node.id, { name: e.target.value })
                }}
              />

                <div className="field">
                  <label>描述</label>

                        description: e.target.value,
                      })
                    }
                    rows={3}
                  />
                </div>
                <div className="field">
                  <label>晋升条件</label>
                  <input

                      updateLevel(system.id, node.branchId, node.id, {
                        promotionCondition: e.target.value,
                      })

                  />

              <div className="field">

                <textarea

                    if (nodeType === 'system') updateSystem(node.id, { description: e.target.value })
```

## 九、性能优化策略

| 优化点 | 策略 | 预期效果 |
|--------|------|----------|
| 树形渲染 | React Arborist 虚拟滚动 | 1000+ 节点流畅 |
| 详情面板 | 懒加载非活动 Tab 内容 | 打开速度 < 200ms |
| 数据更新 | 局部更新，避免全树重渲染 | 操作即时反馈 |
| 搜索过滤 | 100ms debounce | 搜索无卡顿 |
| 持久化 | IndexedDB 分表存储 | 读写 < 500ms |

## 十、风险分析

| 风险项 | 影响 | 概率 | 缓解措施 |
|--------|------|------|----------|
| React Arborist 拖拽与虚拟滚动冲突 | 中 | 低 | 测试验证，必要时降级为非虚拟滚动 |
| 大型体系（100+ 节点）性能下降 | 中 | 中 | 虚拟滚动 + 按需加载 |
| 规则冲突检测延迟高 | 中 | 低 | 后台异步检测 |
| 年表与体系数据关联复杂 | 低 | 中 | 简化关联，仅记录 ID 引用 |

```typescript
/** AI 创建体系提示词 */
export const createSystemPrompt = (description: string) => `
你是一位专业的世界观设定师。请根据用户的描述，生成一个完整的体系结构。

用户描述：${description}

要求：
1. 推断体系类型（力量/社会/经济/道具/知识/生物）
2. 设计合理的分支结构
3. 每个分支设计 3-7 个等级
4. 每个等级包含：名称、描述、晋升条件、能力列表、限制规则

返回 JSON 格式：

  "name": "体系名称",

  "type": "体系类型",
  "branches": [

        {

          "name": "等级名称",

${chapterContent}

1. 检查是否违反任何硬性规则
```
