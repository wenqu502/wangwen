## 一、需求背景与范围界定

### 1.1 PRD 原文需求摘录

根据《PRD-网文创作辅助工具-v3.0.md》：

> **核心理念**：AI 对话是核心交互方式，可视化画布是展示容器。
>
> **主界面布局**：
> - 左侧：主内容区域（可视化画布 / 卡片列表）
> - 右侧：AI 对话面板（常驻）
>
> **AI 对话面板**：
> - 顶部：AI 助手标题 + 设置按钮
> - 中部：对话历史消息列表
> - 底部：输入框 + 快捷指令按钮
>
> **多轮对话上下文记忆**：
> - 记住当前作品的全部设定
> - 用户最近的 20 轮对话历史
> - 支持上下文延续指令（"改一下""再加一个""把它删掉"）

根据《PRD-网文创作辅助工具-v2.0.md》补充：

> - 流式响应（打字机效果）
> - 快捷指令（生成角色/续写剧情/梳理关系）
> - AI 消息类型：文本/建议/操作按钮/卡片预览

### 3.1 自研方案（Custom React + Tailwind）

| 项 | 内容 |
|----|------|
| 技术栈 | React 18 + TypeScript + Tailwind CSS |
| 定位 | 完全自定义对话面板 |
| 优势 | 完全可控，无外部依赖 |

**核心实现要点**

1. **消息列表**：
   - 使用 `react-virtualized` 或 `react-window` 虚拟滚动
   - 消息项高度固定或动态测量
   - 自动滚动到底部（新消息到达时）

2. **流式响应**：
   - 使用 `EventSource` (SSE) 或 `fetch` + `ReadableStream`
   - 逐字追加到消息内容
   - 打字机效果通过定时器控制

3. **消息类型渲染**：
   - 文本消息：Markdown 渲染（`react-markdown`）
   - 建议消息：可点击的建议卡片
   - 操作按钮：内联按钮组
   - 卡片预览：角色卡片/剧情节点预览

| 输入框 | 自定义 textarea | 低 |

## 四、综合评分与选型决策

### 4.1 评分矩阵

| 维度（权重） | 自研方案 | Vercel AI SDK | chat-ui-kit | react-chatbot-kit | LangChain.js |
|-------------|----------|---------------|-------------|-------------------|--------------|
| 技术栈匹配度（25%） | 5 | 5 | 4 | 4 | 3 |
| UI 自定义能力（25%） | 5 | 4 | 3 | 3 | 1 |
| 流式响应支持（20%） | 3 | 5 | 2 | 2 | 4 |
| 性能表现（15%） | 4 | 4 | 3 | 3 | 3 |
| 社区活跃度（10%） | 3 | 5 | 3 | 2 | 5 |
| 许可证（5%） | 5 | 5 | 5 | 5 | 5 |
| **加权总分** | **4.25** | **4.65** | **3.05** | **2.75** | **3.15** |

**评分细则**

- **Vercel AI SDK（4.65/5）**：
  - 流式响应开箱即用，大幅简化开发
  - React Hooks API 现代
  - 社区非常活跃
  - UI 需自研，但 SDK 提供了状态管理基础

- **自研方案（4.25/5）**：
  - 完全可控，但流式响应需自行实现
  - 开发工作量大

- **chat-ui-kit（3.05/5）**：
  - UI 组件丰富，但流式响应不支持
  - 样式定制受限

- **LangChain.js（3.15/5）**：
  - 后端逻辑强大，但无 UI
  - 可作为后端补充

### 4.2 选型结论

**主选方案：Vercel AI SDK + 自研 UI**

理由：
1. **流式响应**：`useChat` Hook 内置 SSE 支持，大幅简化开发
2. **状态管理**：自动管理消息列表、加载状态、错误处理
3. **现代 React**：Hooks API，TypeScript 原生支持
4. **社区活跃**：Vercel 背书，持续迭代
5. **UI 自研**：结合 Tailwind CSS 完全自定义样式，匹配织文设计规范

**后端补充：LangChain.js**

用于：
- 对话上下文记忆管理
- 多模型切换
- 复杂链式调用（如检索 + 生成）

```typescript
  setInput: (input: string) => void
  appendInput: (text: string) => void

  sendMessage: (content: string) => Promise<void>

  // 重试
  retryMessage: (messageId: string) => Promise<void>
}

  immer(
    persist(

        messages: [],
        input: '',
        isLoading: false,
        error: null,
        activeCommands: [
          { id: 'gen-char', label: '生成角色', prompt: '帮我生成一个角色：' },
          { id: 'gen-plot', label: '续写剧情', prompt: '帮我续写剧情：' },
          { id: 'check-relation', label: '梳理关系', prompt: '帮我梳理人物关系' },

        // 消息操作
        addMessage: (message) =>
          set((state) => {
            state.messages.push(message)
          }),

        updateMessage: (id, updates) =>
          set((state) => {
            const msg = state.messages.find((m) => m.id === id)
            if (msg) {
              Object.assign(msg, updates)
            }
          }),

        removeMessage: (id) =>
          set((state) => {
            state.messages = state.messages.filter((m) => m.id !== id)
          }),

        clearMessages: () =>
          set((state) => {
            state.messages = []
          }),

        // 输入框
        setInput: (input) =>
          set((state) => {
            state.input = input
          }),

        appendInput: (text) =>
          set((state) => {
            state.input += text
          }),

        // 加载状态
        setLoading: (isLoading) =>
          set((state) => {
            state.isLoading = isLoading
          }),

        // 错误
        setError: (error) =>
          set((state) => {
            state.error = error
          }),

        // 流式响应
        appendToMessage: (messageId, text) =>
          set((state) => {
            const msg = state.messages.find((m) => m.id === messageId)
            if (msg && msg.content[0]?.type === MessageContentType.TEXT) {
              msg.content[0].text = (msg.content[0].text || '') + text
            }
          }),

            const msg = state.messages.find((m) => m.id === messageId)
            if (msg) {
              msg.isStreaming = false
            }
          }),

        // 快捷指令
        setActiveCommands: (commands) =>
          set((state) => {
            state.activeCommands = commands
          }),

        // 发送消息
        sendMessage: async (content) => {
          const { addMessage, setLoading, setError } = get()

          // 添加用户消息
          const userMessage: ChatMessage = {
            id: `msg_${Date.now()}`,
            role: MessageRole.USER,
            content: [{ type: MessageContentType.TEXT, text: content }],
            timestamp: new Date().toISOString(),
          }
          addMessage(userMessage)

          setLoading(true)
          setError(null)

          try {
            // 调用 AI API（流式响应）
            const response = await fetch('/api/chat', {
              method: 'POST',
              headers: { 'Content-Type': 'application/json' },
              body: JSON.stringify({ message: content }),
            })

            if (!response.ok) {
              throw new Error('发送失败')
            }

            // 创建 AI 消息占位
            const aiMessageId = `msg_${Date.now()}_ai`
            addMessage({
              id: aiMessageId,
              role: MessageRole.AI,
              content: [{ type: MessageContentType.TEXT, text: '' }],
              timestamp: new Date().toISOString(),
              isStreaming: true,
            })

            // 读取流式响应
            const reader = response.body?.getReader()
            if (reader) {
              const decoder = new TextDecoder()
              while (true) {
                const { done, value } = await reader.read()
                if (done) break
                const chunk = decoder.decode(value, { stream: true })
                get().appendToMessage(aiMessageId, chunk)
              }
            }

            get().finishStreaming(aiMessageId)
          } catch (err) {
            setError(err instanceof Error ? err.message : '未知错误')
          } finally {
            setLoading(false)
          }
        },

        // 重试
        retryMessage: async (messageId) => {
          const { messages, sendMessage } = get()
          const msg = messages.find((m) => m.id === messageId)
          if (msg?.role === MessageRole.USER) {
            const text = msg.content[0]?.text
            if (text) {
              await sendMessage(text)
            }
          }
        },
      }),
      {
        name: 'wangwen-chat',
        partialize: (state) => ({
          messages: state.messages,
          activeCommands: state.activeCommands,
        }),
      }
    )
  )
)
```

### 7.4 输入框

```tsx
// src/modules/ai-chat/components/ChatInput.tsx

import React, { useState, useRef } from 'react'
import { useChatStore } from '../store'

interface ChatInputProps {
  disabled?: boolean
}

export const ChatInput: React.FCC<ChatInputProps> = ({ disabled }) => {
  const { input, setInput, sendMessage, isLoading } = useChatStore()
  const textareaRef = useRefRef<HTMLTextAreaElement>(null)

  const handleKeyDown = (e: React.KeyboardEvent) => {
    if (e.key === 'Enter' && !e.shiftKey) {
      e.preventDefault()
      handleSend()
    }
  }

  const handleSend = () => {
    if (!input.trim() || isLoading) return
    sendMessage(input.trim())
    setInput('')
    if (textareaRef.current) {
      textareaRef.current.style.height = 'auto'
    }
  }

  const handleInputChange = (e: React.ChangeEventEvent<HTMLTextAreaElement>) => {
    setInput(e.target.value)
    // 自动调整高度
    e.target.style.height = 'auto'
    e.target.style.height = `${Math.min(e.target.scrollHeight, 120)}px`
  }

  return (
    <div className="chat-input-area">
      <textarea
        ref={textareaRef}
        value={input}
        onChange={handleInputChange}
        onKeyDown={handleKeyDown}
        placeholder="输入消息...（Shift+Enter 换行）"
        disabled={disabled}
        rows={1}
      />
      <button
        className="send-btn"
        onClick={handleSend}
        disabled={!input.trim() || isLoading}
      >
        {isLoading ? '...' : '➤'}
      </button>
    </div>
  )
}
```

## 九、性能优化策略

| 优化点 | 策略 | 预期效果 |
|--------|------|----------|
| 消息列表 | react-window 虚拟滚动 | 1000+ 条消息流畅 |

| 输入框 | 防抖处理 | 减少不必要的渲染 |
| 图片加载 | 懒加载 | 减少初始加载 |
