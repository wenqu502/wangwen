# 网文创作辅助工具 — 产品需求文档 (PRD) v3.0

> **版本**: v3.0  
> **日期**: 2026-05-03  
> **定位**: AI-first 的一站式网文创作辅助 SaaS 工具  
> **核心信条**: 用对话完成创作，用画布确认结构  
> **形态**: Web 应用（桌面端为主）

---

## 一、产品概述

一款以 **AI 对话为核心交互方式** 的网文创作辅助工具。用户通过自然语言与 AI 助手交流，完成角色创建、剧情规划、关系梳理、体系设计等创作任务。可视化画布（角色卡片、剧情树、关系图等）作为 **AI 生成内容的展示容器和手动微调工具**，而非从零手动搭建的编辑器。

**一句话定义**：你说话，AI 画图；你看图，AI 改图。

**核心价值主张**：
- 不让作者因为「忘记了自己写过什么」而写崩
- 不让作者因为「不知道怎么推进剧情」而卡文
- 不让作者因为「手动维护设定表格」而烦躁

## 二、目标用户

| 用户类型 | 特征 | 与工具的交互方式 |
|----------|------|------------------|
| 职业/半职业网文作者 | 日更压力大，长篇连载 | 用 AI 快速生成角色、续写剧情、校验一致性 |
| 新人写手 | 缺乏大纲能力，容易写崩 | 用 AI 引导式对话完成世界观搭建和剧情规划 |
| 同人创作者 | 基于原作二次创作 | 用 AI 导入原作信息后生成 OC 和剧情线 |
| 剧本杀/OC 创作者 | 注重人物关系与设定 | 用 AI 生成复杂关系网络和设定体系 |

### 3.2 交互优先级

| 优先级 | 交互方式 | 说明 |
|--------|----------|------|
| **P0** | **自然语言对话** | 用户描述意图，AI 执行操作。这是主要交互方式。 |

| **P2** | **表单填写** | 仅在 AI 对话后需要精确校正时，才展示表单供用户手动编辑。 |

## 四、全局 AI 创作助手

一个始终驻留在右侧边栏的 AI 对话面板，是用户与工具交互的**主入口**。

### 4.1 能力范围

| 能力 | 说明 |
|------|------|
| 意图理解 | 理解用户的自然语言创作意图，转化为结构化操作 |
| 内容生成 | 基于用户描述，生成角色、剧情、关系、体系等内容 |
| 上下文记忆 | 记住当前作品的全部设定，确保生成内容前后一致 |
| 多轮对话 | 支持"改一下""再加一个""把它删掉"等上下文延续指令 |
| 主动建议 | 在检测到潜在问题时（如角色崩坏风险、伏笔未回收），主动提醒 |

### 4.2 对话上下文管理

AI 助手需要维护以下上下文信息：

```
当前作品全部角色档案
当前作品全部剧情节点
当前作品全部关系数据
当前作品全部体系设定
当前作品全部灵感便签
用户最近的 20 轮对话历史
```

### 5.2 剧情分支树（Plot Branch Tree）

**核心理念**：用户用自然语言描述故事脉络，AI 生成树形结构。

#### 5.2.1 AI 对话式生成骨架

```
用户：我想写一个悬疑复仇故事，女主发现闺蜜是被现任男友害死的，
      决定假装不知道接近他收集证据，最后揭露真相

AI：已为您生成剧情骨架，包含 18 个节点：
    
    【主线脉络】
    节点1: 开篇 - 女主得知闺蜜死讯，发现疑点
    节点2: 调查 - 发现闺蜜死前与男友有关联
    节点3: 决定 - 制定复仇计划，接近男友
    节点4~8: 隐忍阶段 - 收集证据，步步为营
    节点9: 转折 - 发现真相比想象中更黑暗
    节点10~14: 高潮阶段 - 设局反击
    节点15: 揭露 - 在关键场合公开证据
    节点16: 结局 - 正义得以伸张，女主走出阴影
    
    【分支点】
    分支A(节点6): 男友发现女主在调查 → 危险升级
    分支B(节点11): 警方介入 → 合作 or 独自行动
    
    画布已更新。您可以说"加一条支线""把节点5删掉"
    "在节点3后面再加一个情感铺垫"等指令继续调整。
```

#### 5.2.2 AI 对话式修改

```
用户：在节点5后面加一个情感冲突，让女主差点暴露
AI：已插入新节点「节点5.5：意外暴露危机」
    内容：女主在男友电脑中发现关键证据时，男友突然回家。
    她被迫假装浏览购物网站，心跳如鼓。
    
    同时，我在分支A中增加了一个回收点：
    这个"差点暴露"的经历，让女主后续行动更加谨慎。

用户：那个分支B我不要了，删掉
AI：已删除分支B（节点11及后续分支节点）。
    相关节点的状态已标记为"废弃"，但数据保留，随时可恢复。
```

#### 5.2.3 可视化展示与微调

AI 生成剧情树后，在左侧画布展示：

- **节点卡片**：显示标题、摘要、状态（已写/待写/进行中）
- **连线**：实线表示主线，虚线表示分支
- **颜色**：已写（绿色）、进行中（蓝色）、待写（灰色）、废弃（浅色）
- **用户可执行的操作**：
  - 拖拽节点调整位置
  - 点击节点查看/编辑详情
  - 拖拽连线调整分支关系
  - 右键菜单：删除、添加子节点、标记状态

#### 5.2.4 AI 伏笔追踪

```
【自动检测】

· 突然出现的物品/人物
· 角色说出的意味深长的话
· 被刻意忽略的细节

【AI 提醒示例】
AI：🔮 检测到潜在伏笔
    您在第12章写道：「她注意到男友钱包里有一张陌生女人的照片，
    但他迅速收了起来。」
    
    我已将其标记为伏笔「陌生女人照片」。
    是否需要我追踪这条线索的回收？
    
    [追踪] [忽略] [标记为关键线索]

【未回收提醒】
AI：⚠️ 伏笔回收提醒
    伏笔「陌生女人照片」已埋设 8 章未回收。
    当前剧情节点：第20章。
    
    建议回收位置：
    A) 在第22章揭示照片女人是男友的前女友

### 5.4 体系管理工具（System Builder）

**核心理念**：完全抽象的体系设计工具，不限定任何题材。

#### 5.4.1 什么是"体系"

「体系」是作品中任何**分层分级、有规则约束的结构化设定**。包括但不限于：

| 类型 | 示例 |
|------|------|
| 力量体系 | 修炼境界、魔法等级、异能分类 |
| 社会体系 | 贵族爵位、官僚等级、帮派辈分 |
| 经济体系 | 货币兑换、资源稀有度、物价体系 |
| 道具体系 | 装备品质、武器分级、丹药品阶 |
| 知识体系 | 学科分类、技能树、科技树 |
| 生物体系 | 物种等级、进化链、食物链 |

#### 5.4.2 AI 对话式创建体系

```
用户：我想设计一个魔法体系，分元素系和召唤系，

## 六、AI 智能校验中心

所有校验功能均由 AI 自动执行，无需用户手动触发。

### 6.1 人设一致性校验

| 触发时机 | 校验内容 | AI 反馈方式 |
|----------|----------|-------------|
| 修改角色档案后 | 扫描已写章节，标记冲突 | 弹出冲突列表，附带修改建议 |
| 撰写新章节时（实时） | 检测角色言行是否符合档案 | 段落旁标记⚠️，侧边栏显示详情 |
| 提交章节时（批量） | 全章一致性评分 | 生成一致性报告，列出风险点 |

### 6.2 伏笔追踪

| 功能 | AI 行为 |
|------|---------|
| 自动识别 | 扫描正文，识别潜在伏笔并询问用户确认 |
| 自动追踪 | 记录伏笔状态（已埋/已回收/遗忘） |
| 回收提醒 | 超过设定阈值未回收时主动提醒 |
| 回收建议 | 根据剧情走向建议最佳回收时机 |

### 6.3 规则冲突检测

| 功能 | AI 行为 |
|------|---------|
| 规则理解 | 将自然语言描述的规则解析为可执行条件 |
| 实时检测 | 在写作过程中实时扫描冲突 |
| 批量检测 | 支持对已完成章节进行全盘扫描 |
| 冲突解决 | 提供多种解决方案供用户选择 |

## 七、明确不做（产品边界）

以下功能**明确不在本工具范围内**：

| 不做的事 | 原因 |
|----------|------|
| ❌ 接入网文平台的读者评论 | 这是创作辅助工具，不是读者互动平台 |
| ❌ 接入网文平台的弹幕数据 | 同上，且弹幕数据对创作辅助无直接价值 |
| ❌ 作品数据统计（阅读量、收藏数） | 不接入平台数据，无法获取 |
| ❌ 网文平台热点/市场趋势分析 | 不做数据分析平台，专注创作本身 |
| ❌ 自动发布到网文平台 | 不做发布渠道，只产出创作资料 |
| ❌ 读者画像分析 | 没有读者数据，不做此功能 |
| ❌ 收益/稿费计算 | 与平台无关，不做财务功能 |

**一句话总结**：我们只帮作者「写出好故事」，不帮作者「运营好故事」。

## 十一、交付验收标准

### 11.1 核心交互验收

- [ ] 用户可以通过纯对话完成：创建角色、生成剧情树、梳理关系、设计体系
- [ ] AI 生成的内容在可视化区域正确展示
- [ ] 用户可以在可视化区域做拖拽微调
- [ ] AI 对话支持上下文延续（"改一下""再加一个""删掉"）
- [ ] AI 能主动检测并提醒潜在问题（伏笔、冲突、不一致）

### 11.2 功能验收

- [ ] 角色创建：AI 根据一句话描述生成完整档案
- [ ] 剧情生成：AI 根据梗概生成可调整的剧情树
- [ ] 关系梳理：AI 根据角色档案自动推断关系
- [ ] 体系设计：AI 根据描述生成抽象化的分级体系
- [ ] 灵感提取：AI 自动从对话中捕获灵感片段
- [ ] 伏笔追踪：AI 自动识别伏笔并提醒回收
- [ ] 规则检测：AI 理解规则并检测冲突

### 11.3 性能验收

- [ ] AI 对话响应时间 < 5s（常规生成））< 15s（复杂生成如剧情树）
- [ ] 可视化画布 50 个节点时拖拽流畅
- [ ] 首次加载 < 3s

## 十二、附录：开发决策记录

```
1. [AI-First] 对话是主要交互，可视化是展示和微调工具 —— 用户明确要求
2. [抽象化] 体系管理不限定任何题材（修炼/魔法/货币均可）—— 用户明确要求
3. [本地优先] 暂不接入网文平台数据（评论/弹幕/统计均不做）—— 用户明确要求
4. [AI 全覆盖] 每个功能模块都深度集成 AI，不是可选功能 —— 用户明确要求
5. [隐私优先] 作品数据本地存储，不上传云端 —— 网文作者对稿件敏感
6. [不越界] 只帮创作，不帮运营（不做市场分析、不做发布渠道）—— 产品边界清晰
```
