# 织文 (WangWen) — AI 协作规范

> 如果你是 AI Agent，请先阅读此文件，再开始任何操作。

## 项目定位

织文是一个 **AI-first 的网文创作辅助工具**。所有功能都围绕 AI 对话交互设计，可视化画布仅作为展示和微调容器。

**核心信条**：用对话完成创作，用画布确认结构。

## 如果你是 AI Agent

### 开发原则

1. **AI-first 优先**：每一个新功能都要思考「如何通过自然语言交互完成」，而不是「如何做一个表单」。
2. **模块自治**：每个模块（角色/剧情/关系/体系/灵感）有独立的类型定义、状态管理和 AI 提示词。
3. **对话驱动状态**：AI 不操作 DOM，只生成结构化数据，由 React 响应式更新画布。
4. **本地优先**：所有数据存储在浏览器本地（IndexedDB），不上传云端。
5. **隐私保护**：用户的创作数据不出本地，AI API Key 由用户自行配置。

### 代码规范

- **TypeScript**：所有代码必须严格类型化，`any` 需要注释说明原因。
- **命名**：
  - 组件：PascalCase（如 `ChatPanel.tsx`）
  - 函数/变量：camelCase
  - 类型/接口：PascalCase（如 `Character`）
  - 常量：UPPER_SNAKE_CASE
- **文件组织**：一个文件一个职责。组件、Hook、Store 分离。
- **状态管理**：使用 Zustand + Immer，避免直接修改状态。
- **样式**：Tailwind CSS 原子类为主，自定义样式用 CSS Variables。

### 提交规范

参考 [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: 新功能
fix: Bug 修复
docs: 文档更新
style: 代码格式（不影响逻辑）
refactor: 重构（非 feat/fix）
perf: 性能优化
test: 测试相关
chore: 构建/工具相关
```

### PR 要求

- 每次只解决一个问题
- 修改前先在本地验证功能正常
- 提交前运行 `pnpm lint`
- PR 描述要说明修改原因和验证方式

## 项目边界（明确不做）

- ❌ 不接入网文平台的读者评论/弹幕/统计数据
- ❌ 不做自动发布到网文平台
- ❌ 不做读者画像分析
- ❌ 不做收益/稿费计算
- ❌ 不做市场热点趋势分析

**一句话**：我们只帮作者「写出好故事」，不帮作者「运营好故事」。
